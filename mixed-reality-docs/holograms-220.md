## <a name="chapter-2---spatial-sound-and-interaction"></a>2 장-공간 소리 및 상호 작용

### <a name="objectives"></a>목표

* 소리를 사용 하 여 홀로그램 현실성을 향상 시킵니다.
* 소리를 사용 하 여 사용자의 게이즈를 지시 합니다.
* 소리를 사용 하 여 제스처 피드백을 제공 합니다.

### <a name="part-1---enhancing-realism"></a>1 부-현실성 향상

#### <a name="key-concepts"></a>주요 개념

* 홀로그램 소리 spatialize 합니다.
* 사운드 소스를 홀로그램의 적절 한 위치에 배치 되어야 합니다.

소리에 대 한 적절 한 위치를 홀로그램 종속 것입니다. 예를 들어를 홀로그램 사람이 인 경우 사운드 원본 입 및 feet 없습니다 근처에 있어야 합니다.

#### <a name="instructions"></a>지침

다음 지침을 홀로그램 spatialized 소리를 연결 됩니다.

* 에 **계층 구조** 패널에서 확장 **HologramCollection** 선택한 **P0LY**합니다.
* 에 **검사기** 패널의 **AudioSource**, 옆에 원을 클릭 **AudioClip** 선택한 **PolyHover** 팝업에서.
* 옆에 원을 클릭 **출력** 선택한 **SoundEffects** 팝업에서.

Unity를 사용 하는 프로젝트 데시벨 **AudioMixer** 소리의 그룹에 대 한 사운드 수준을 조정 하는 사용 하도록 설정 하려면 구성 요소입니다. 소리가 그룹화 이러한 방식으로 각 사운드의 상대 볼륨 유지 관리 하는 동안 전체 볼륨을 조정할 수 있습니다.

* 에 **AudioSource**를 확장 하 고 **3D 소리 설정**합니다.
* 설정할 **doppler가 수준** 하 **0**합니다.

0 개 사용 하지 않도록 설정 변경 (사용자나를 홀로그램 중 하나) 동작을 야기 하는 피치에 doppler가 수준을 설정 합니다. Doppler가 전형적인 예로 빠르게 움직이는 차량 합니다. 자동차 고정 수신기 다가오면 엔진의 피치 증가 합니다. 수신기를 전달할 때 피치 거리를 사용 하 여 낮춰 줍니다.

### <a name="part-2---directing-the-users-gaze"></a>2 부-사용자의 게이즈를 전달 합니다.

#### <a name="key-concepts"></a>주요 개념

* 중요 한 홀로그램을 강조할 소리를 사용 합니다.
* 돌릴 눈 같아야 여기서 직접 데 도움이 됩니다.
* 뇌에 몇 가지 학습된 기대 합니다.

학습된 예상의 예로 새 일반적으로 사용자의 머리 위의. 사용자에서 bird 소리를 수신 하는 경우 해당 초기 반응은 조회 하는 것입니다. 사용자 아래 bird 배치의 소리를 조회할 필요가 기대에 따라 홀로그램을 찾을 수 없거나 올바른 방향을 향하고에 발생할 수 있습니다.

#### <a name="instructions"></a>지침

다음 지침을를 홀로그램 찾으려고 소리를 사용할 수 있도록 백그라운드에서 숨기려면 P0LY를 사용 합니다.

* 만들기 및 구성 된 **AudioSource**합니다.
* 위치는 **AudioSource** 적절 한 위치의 **GameObject**합니다.
* 재생 합니다 **AudioClip** 제스처와 연결 합니다.

1. Unity에서 선택 **파일 > 빌드 설정**합니다.
2. **빌드**를 클릭합니다.
3. 한 번 클릭 합니다 **앱** 폴더입니다.
4. 키를 눌러 **폴더 선택**합니다.

도구 모음에서 "릴리스", "x86" 또는 "x64" 및 "원격 장치" 라는 있는지 확인 합니다. 그렇지 않은 경우이 Visual Studio의 코딩 인스턴스입니다. 앱 폴더에서 솔루션을 다시 엽니다 해야 합니다.

* 메시지가 표시 되 면 프로젝트 파일을 다시 로드 합니다.

## <a name="chapter-4---sound-design"></a>4 장-적절 하 게 디자인

### <a name="objectives"></a>목표

### <a name="part-1---sound-and-experience-design"></a>1 부-소리 및 환경 디자인

이 섹션에서는 키 소리 및 환경 디자인 고려 사항 및 지침을 설명 합니다.

#### <a name="normalize-all-sounds"></a>모든 소리를 정규화 합니다.

이 시간이 오래 걸릴 수 있는 소리 당 볼륨 수준에 맞게 특수 사례 코드에 대 한 필요가 없습니다를 쉽게 사운드 파일을 업데이트 하는 기능을 제한 합니다.

#### <a name="design-for-an-untethered-experience"></a>제한 되지 않는 환경 디자인

HoloLens에 완전히 포함 된, 클라우드가 holographic 컴퓨터입니다. 사용자를 이동 하는 동안 경험을 사용 합니다. 에 오디오 믹스를 따라 이동 하 여 테스트 해야 합니다.

#### <a name="emit-sound-from-logical-locations-on-your-holograms"></a>소리에 홀로그램의 논리적 위치에서 내보내기

실제 상황에서 dog 꼬리에서 짖는 하지 않습니다 및 사람이 음성 자신의 피트에서 제공 되지 않습니다. 소리 내보낼에서 예기치 않은 부분에 제공 하지 않아도 됩니다.

작은 홀로그램 기 하 도형 중심에서 내보낼 소리 할 적절 한 것입니다.

#### <a name="familiar-sounds-are-most-localizable"></a>친숙 한 소리는 지역화할 수 있는 가장

인간의 음성 및 음악은 매우 쉽게 지역화할 수 있습니다. 사용자 이름, 호출 하는 경우에 매우 정확 하 게 결정할 방법에서 제공 하는 음성 방향을 멀리 수 있습니다. 짧은, 알 수 없는 소리 지역화 하기는 어렵습니다.

#### <a name="be-cognizant-of-user-expectations"></a>사용자 기대 한다는 점을 인식

소리의 위치를 식별 하는 데에서 일부를 담당 하는 경험 합니다. 사람의 음성 특히 쉽게 지역화할 수 없는 이유는 하나의 이유입니다. 소리를 배치 하는 경우 사용자의 학습 기대 알아야 할 것입니다.

예를 들어 누군가가 시도해볼 bird 노래 하는 경우 일반적으로 조회 하는, 시야 (공중 또는 트리에서) 위에 새 경향이으로 합니다. 소리의 올바른 방향으로 설정 되지만 잘못 된 세로 방향에서 모양과 홀로그램을 찾을 수 없는 경우 혼동 되거나 불편 될 사용자에 대 한 일반적이 지 않은 것입니다.

#### <a name="avoid-hidden-emitters"></a>숨겨진된 송신기 방지

실제 상황에서 소리를 구하면에서는 일반적으로 식별할 수 소리를 표시 하는 개체입니다. 이 경험에 변수에 있어야 합니다. 소리를 듣고, 소리의 원본 위치에서 알고 있어야 하는 사용자에 대 한 매우 혼란을 줄 수 하 고 개체를 참조 하지 못할 수 있습니다.

이 지침을 몇 가지 예외가 있습니다. 예를 들어 crickets 필드에 같은 앰비언트 소리 표시 되지 않아도 됩니다. 수명 환경을 볼 필요 없이 이러한 소리의 소스와 경험을 제공 합니다.

오디오 데이터는 일반적인 샘플 속도 (44.1 및 48 kHz)에서 특히 큰 수 있습니다. 일반 규칙은 응용 프로그램 메모리 사용량을 줄이기 위해 5 ~ 10 초 이상 오디오 파일을 스트리밍할 수 해야 합니다.

Unity에서 파일의 가져오기 설정에서 스트리밍에 대 한 오디오 파일을 표시할 수 있습니다.

#### <a name="instructions"></a>지침

Unity의 매개 변수를 구성 하는 이전 설정이 **AudioChorusFilter** 사용자의 음성에 다양 한 기능을 추가 하는 데 사용 합니다.

* 설정할 **비율 Decay** 하 **0.2**합니다.
* 설정할 **원래 소리 볼륨** 하 **0**합니다.
