# <a name="case-study---capturing-and-creating-content-for-holotour"></a>사례 연구-캡처하고 HoloTour 콘텐츠 만들기

Microsoft HoloLens 대 한 HoloTour 몰입 감이 뛰어난 3D 개인 tours 세계 아이콘 위치를 제공합니다. 디자이너, 아티스트, 생산자, 오디오 디자이너 및 개발자가 발견이 프로젝트에서 작업으로 잘 알려진 위치의 실제 convincingly 3D 렌더링을 만드는 데 독창적이 고 기술적 없의 고유한 혼합을 걸립니다.

## <a name="the-tech"></a>기술 지원 담당자

HoloTour를 사용 하 여 세계에서 가장 놀라운 대상의 일부를 방문 하는 데 수 있도록 하려는-같은 합니다 [마추 픽추 중 버려진](https://en.wikipedia.org/wiki/Machu_Picchu) 페루 또는 최신 일 [Piazza Navona](https://en.wikipedia.org/wiki/Piazza_Navona) 이탈리아에서-부터 자신의 거실 합니다. 필자의 팀 있게 HoloTour 목표 "실제로 여기에 있는 것 처럼 느껴질 수 수 있도록 합니다." 사진 또는 비디오 이외의 추가 해야 하는 환경. 고유한 표시, 추적 및 HoloLens의 오디오 기술을 활용 하 여 있습니다 다른 위치로 전송 거의 없습니다 것 라고 생각 했습니다. 캡처 위권, 소리 및 3 차원 기 하 도형 방문 하 고 다시는 앱 내의 모든 위치 해야 합니다.

이 작업을 수행 하기 위해 360 ° 카메라 했던 방향 오디오 캡처를 사용 하 여 장치입니다. 해당 하는 카메라는 HoloLens에서 재생 하는 경우에 관하여 보입니다이 카메라 연결 아티팩트를 최소화 하기 위해 서로 가까이 배치 해야 합니다. 매우 높은 해상도 캡처하려면 필요 합니다. 전체 구형 검사, 수평을 따라 뿐 아니라 하지만 위와 아래도 싶었습니다. Rig 전 세계 모든 것을 가져올 수 있습니다 있도록 이식 가능 해야 합니다. 사용 가능한 기본 제공 옵션을 평가 하 고 실현 단순히 당시에 이러한 확인, 비용, 또는 크기 때문 – 우리의 비전을 실현 하는 것으로 충분 합니다. 요구 사항 충족 하는 카메라 rig를 찾을 수 없습니다, 하나의 직접 빌드는 했습니다.

### <a name="capturing-the-video"></a>비디오를 캡처

몇 가지 간단한 사항을 캡처하는 동안 수행할 후 처리 훨씬 더 쉽게 만들 수 있습니다. 예를 들어, 있습니다를 함께 붙이기 위한 여러 카메라에서 이미지, 때마다 얻게 visual 아티팩트를 사용 하 여 각 카메라 약간 다른 보기에 있으므로. 좀 더 자세히 개체는 더 큰 카메라의 차이 뷰 및 클수록 아티팩트 연결 됩니다. 문제를 시각화 하는 간편한 방법은 다음과 같습니다: 얼굴 앞에 있는 엄지 손가락을 지연 하 고 하나의 눈을 사용 하 여 살펴봅니다. 이제 눈을 전환 합니다. 백그라운드 기준으로 이동 하 여 엄지 단추 표시 되는지 볼 수 있습니다. 얼굴에서 추가로 엄지 손가락을 보유 하 고 실험을 반복에 thumb 작은 이동할 표시 됩니다. 명백한 이동 연결 문제에 비슷합니다. 발생 한 큰 문제점: 이 카메라와 같은 자신만 작은 간격으로 구분 하는 때문에 동일한 이미지를 정확 하 게 표시 되지 않습니다.

사후 처리에서이 해결 하는 것 보다 filming 하는 동안 최악의 아티팩트를 방지 하기 위해 쉽게 이기 때문에 사람과 작업 자세히 개체를 결합할 필요가 제거 수 하기 위해 카메라에서 멀리 떨어진 유지 노력 합니다. 이 카메라 중심 큰 지우기 유지 관리 했습니다. 해결 하는 동안 가장 큰 과제 중 하나 였습니다 아마도 및 작동 하도록 창의성 발휘 해야 했습니다. 로컬 가이드를 사용 하 여 작업 관리 관중에 엄청난 도움말 했지만 사용 하 여 서명 하는 것도 찾을 수-경우에 따라 작은 콘 또는 bean 모음 및-우리의 filming 공간 표시를 합리적으로 효과적으로 해야 하는 각 위치에서 카메라를 짧은 시간 가져오기 때문에 특히 합니다. 종종 좋은 캡처 가져오기에 대 한 가장 좋은 방법은만 아침 사람들이 나타난 하기 전에 초기에 도착 했습니다.

최적화 및 압축 후 단일 비디오 지역의 예입니다. 


디코딩 비디오 보기 적극적으로 되지 않아도 다른 방법을 사용 했습니다. HoloTour에 있는 동안 언제 든 전체 장면의 일부를 볼 수 있습니다. 에서는 비디오를 내에서 또는 필드의 보기 (FOV) 외부 곧만 디코딩합니다. 머리를 회전 하는 대로 프로그램 FOV에서 이제는 더 이상 내는 재생을 중지 하는 지역 비디오의 재생을 시작 합니다. 이 발생 하는, 있지만 신속 하 게 반환 하는 경우 비디오를 사용 하려면 두 번째 확인할 수 있습니다에 대부분의 사람들이 알지 못하는-그동안 준비가 되 면 다음 페이드는 비디오를 정적 이미지가 표시 됩니다.

이 전략을 사용할 수 있도록 광범위 한 비디오 재생 시스템을 개발 했습니다. 있도록 전환 비디오 매우 효율적인 하위 수준 재생 코드를 최적화 했습니다. 또한 언제 든 지 비디오를 신속 하 게 전환할 수 있도록 특별 한 방식으로 비디오를 인코딩 했습니다. 이 재생 파이프라인 단계에서 구현 했습니다 상당한 개발 시간을 소요 되었습니다. 효율성이 떨어지기 간단한 시스템을 사용 하 여 시작 되지만 허용된 디자이너 및 아티스트가 작업 환경 및 느린 최종 품질 표시줄을 제공할 수 있도록 해 주는 보다 강력한 재생 시스템을 개선 합니다. 이 최종 시스템 장면 및 재생 엔진이 모니터 내에서 비디오를 설정 하는 Unity 내에서 만든 사용자 지정 도구를 했습니다.

### <a name="recreating-near-space-objects-in-3d"></a>3d에서 가까운 공간 개체를 다시 만들기

비디오 표시 HoloTour, 대부분이 구성 되지만 Pantheon, 또는 항공 장면을 위한의 stand 뜨거운 공기 풍선 외부 샘 Piazza Navona에서 그리기와 같은 가까운, 표시 되는 3D 개체의 수입니다. 이러한 3D 개체 멀리 휴먼 깊이 perception은 매우 양호를 자세히, 때문에 중요 하지만 매우 좋지 됩니다. 에서는 가져올 수 지금 비디오를 사용 하 여 거리에 있지만 필요한 깊이 주변 개체 해당 공간과 실제로 있는 것 처럼 느낌 경비원 사용자를 허용 하려면. 이 방법은 명언을 남겼지만 표시 될 수 있습니다 자연 기록 박물관에서 유사한-포그라운드에서 실제 조 경, 시설 및 동물 박쥐 갖지만 오목 하 게 백그라운드에서 영리 하 게 마스킹된 매트 그림에 표시 하는 한 모형 그림입니다.

일부 개체는 단순히 3D 자산 생성 하 고 환경을 개선 하기 위해 장면에 추가 합니다. 그리기 및 공기 풍선에서는 촬영 했을 때 존재 하지 않은 때문에이 범주에 속합니다. 게임 자산 마찬가지로 된 필자의 팀에서 3D artist를 만든 하며 질감을 적절 하 게 합니다. 이 장면 위치 근처에 배치 했습니다 및 게임 엔진 렌더링할 수 두 HoloLens 표시 하는 3D 개체로 표시 되도록 합니다.

Pantheon 외부 샘와 같은 다른 자산 위치에서는 촬영 하 비디오 위치에 있지만 비디오에서 이러한 개체를 3D에 존재 하는 실제 개체를 여러 가지 작업을 수행 해야 합니다.

첫째, 각 개체에 대 한 정보를 추가 해야합니다. Filming에 대 한 위치에 있는 동안 팀 캡처된 이러한 개체의 참조 짧은 장면이 라도 여러에 우리는 이미지를 질감을 정확 하 게 다시 자세한 충분 합니다. 팀도 수행 합니다.는 [photogrammetry](https://en.wikipedia.org/wiki/Photogrammetry) 검색, 2D 이미지 중에서 3D 모델을 생성 하는 완벽 하 게 대규모로 개체의 대략적인 모델을 제공 합니다.

우리의 영상 처리에서는 3D 표현을 사용 하 여 나중에 바꿀 개체 비디오에서 제거 됩니다. 3D 자산 photogrammetry 모델을 기반으로 하지만 정리 및 우리의 아티스트도 간소화 합니다. 일부 개체에 대 한 비디오의 일부 사용할 수 있습니다-샘에서 최고 수 위 질감 같은-되지만 샘 대부분이 된 환경의 제한 된 공간에서 경비원 및 깊이 인식할 수 있는 3D 개체입니다. 같이 가까운 공간 개체를 크게 필요 현실성의 의미를 추가 하 고 가상 위치에 사용자를 포함 하는 데 도움이 됩니다.
