## <a name="spatial-coordinate-systems"></a>공간 좌표계

모든 3D 그래픽 응용 프로그램 사용 [데카르트 좌표계](https://docs.microsoft.com/windows/uwp/graphics-concepts/coordinate-systems) 위치 및 렌더링 하는 가상의 세계에 있는 개체의 방향에 대 한 추론 합니다. 이러한 좌표 시스템 설정 개체를 배치 하는 3 개의 수직 축: X, Y 및 Z 축 합니다.

[혼합 현실을](mixed-reality.md), 가상 및 실제 좌표 시스템에 대 한 앱은 이유입니다. Windows 실제에서 실제 의미가 있는 좌표 시스템 호출을 **공간 좌표계**합니다.

공간 좌표계 미터의 좌표 값을 표현합니다. 즉, 개체 2 개 단위 X 중 하나에서 떨어져 배치 Y 또는 Z 축 2m 혼합된 현실에서 렌더링 될 때 서로 외에도 표시 됩니다. 이 기능을 사용 하면 쉽게 실제 대규모로 환경과 개체를 렌더링할 수 있습니다.

일반적으로 데카르트 좌표계는 오른손 또는 왼쪽 일 수 있습니다. Windows에서 공간 좌표계는 항상 오른손, 양수 x 축 상태의 오른쪽에 양의 y-축 (중력 맞게) 요소 및 사용자 쪽 양수 z 축 지점이 있습니다.

두 종류의 좌표 시스템에서 양수 x 축은 오른쪽을 가리키는 양의 y-축 합니다. 양의 z-축 방향으로 또는 사용자 반대쪽을 가리키는 하는 여부는 다릅니다. 왼쪽 또는 오른쪽 방향 및 고 양의 Y 방향을 컬링 X 양수에 사용자의 손가락을 가리키면 가리키는 양의 z-축 방향을 기억할 수 있습니다. 엄지 손가락 지점으로 또는 사용자 반대쪽, 방향은 해당 좌표계에 대 한 양의 z-축 가리키는 방향입니다.

## <a name="building-an-orientation-only-or-seated-scale-experience"></a>한 방향 으로만 이동 가능한 또는 장착 규모 환경 구축

키를 holographic [렌더링](rendering.md) 해당 홀로그램 보기 앱의 해당 예측된 헤드 동작과 일치 시킬 사용자가 이동 하는 대로 각 프레임 변경 됩니다. 빌드할 수 있습니다 **장착 규모 환경** 존중 사용자의 헤드 위치 및 헤드 방향을 사용 하 여 변경 하는 **고정 참조 프레임**합니다.

일부 콘텐츠 헤드 위치 지정 업데이트를 선택한 머리글을 고정 된 작업을 무시 해야 및 사용자 로부터의 거리에 항상. 한 가지 예로 360도 비디오: 사용자가 하는 대로 보기 방향을 변경 해야 하는 경우에 콘텐츠를 기준으로 이동할 뷰 위치에 대 한 효과 손상는 단일 고정된 관점에서 비디오를 캡처하는, 때문에 있습니다. 같은 빌드할 수 있습니다 **방향 으로만 이동 가능한 환경을** 를 사용 하 여는 **연결의 참조 프레임**합니다.

### <a name="stationary-frame-of-reference"></a>고정 참조 프레임

고정 참조 프레임 작동 되도록 사용자 근처 개체의 위치를 최대한 안정적으로 전 세계를 기준으로 사용자의 헤드 위치에서 변경 내용을 유지 하는 동안 제공한 좌표계입니다.

장착 규모와 같은 게임 엔진의 환경에 대 한 [Unity](https://unity3d.com/), 고정 참조 프레임은 엔진의 "세계 원본입니다." 정의 특정 세계 좌표에 배치 되는 개체 동일한 좌표를 사용 하 여 실제 환경에서의 위치를 정의 하려면 고정 참조 프레임을 사용 합니다. 세계에서, 사용자에 설명 하는 것에 유지에 배치 하는 콘텐츠 이라고 **world 잠긴** 콘텐츠입니다.

앱은 일반적으로 만들고 한 고정 참조 프레임 시작 시 사용 하 여 앱의 수명 주기 동안 해당 좌표계입니다. 앱 개발자는 Unity의 콘텐츠는 사용자의 초기 헤드 위치 및 방향 될 원본에 상대적인 배치만 시작할 수 있습니다. 사용자를 새 위치로 이동 하 고 장착 규모 경험을 계속 하려고 하면 하면 둘러싸기의 world 원본 위치에 있습니다.

시간이 지남에 따라 시스템은 사용자의 환경에 대 한 더 학습 하는 대로 해당 결정할 수 있습니다 실제의 다양 한 지점 사이의 거리는 이전에 힘들게 시스템 보다 짧거나 길입니다. 홀로그램 5 미터에 대 한 다양 한 영역을 벗어나는 사용자가 겪고 있는 HoloLens에 앱에 대 한 고정 프레임의 참조를 렌더링 하는 경우 앱 해당 홀로그램의 관찰 된 위치에서 드리프트를 관찰할 수 있습니다. 빌드하는 환경을 사용자 5m 초과 wandering 있으면를 [세계 규모 환경](#building-a-world-scale-experience), 아래 설명 된 대로 홀로그램 안정적이 고 유지 하는 추가 방법 필요 합니다.

### <a name="attached-frame-of-reference"></a>연결 된 프레임의 참조

이러한 경비원을 앱 프레임을 처음 만들 때 정의 된 고정된 머리글을 사용 하 여 사용자와 연결 된 프레임의 참조를 이동 합니다. 이 참조는 프레임 내에 배치 하는 콘텐츠 편안 하 게 확인 수가 있습니다. 이 사용자 관련 방식으로 렌더링 하는 콘텐츠 이라고 **본문 잠긴** 콘텐츠입니다.

전 세계의 경우 헤드셋 알 수 없습니다, 하는 경우 연결된 된 프레임의 참조 홀로그램 렌더링에 사용할 수 있는 유일한 좌표계를 제공 합니다. 따라서 장치의 전 세계에서 찾을 수 없는 사용자에 게 알리고 대체 UI 표시에 대 한 적합 합니다. 장착 규모 또는 더 높은 앱 사용자에 표시 된 것과 비슷한 UI를 사용 하 여 다시 시작 하는 데는 방향 으로만 이동 가능한 대체를 포함 해야 합니다 [혼합 현실 홈](navigating-the-windows-mixed-reality-home.md)합니다.

## <a name="building-a-standing-scale-or-room-scale-experience"></a>고정 크기 조정 또는 방 규모 환경 구축

몰입 형 헤드셋에서 앉아 규모를 초과할를 구축 하는 **고정 규모 환경**를 사용할 수는 **단계에서 참조 하는 프레임을**입니다.

제공 하는 **방 규모 경험**하도록 사용자가 미리 정의 된 5 미터 경계 내에서 경비원을 확인할 수 있습니다 **범위 단계** 도 합니다.

### <a name="stage-frame-of-reference"></a>단계에서 참조 하는 프레임을

사용자 정의 몰입 형 헤드셋을 처음 설정할 때를 **단계**, 혼합 현실을 경험할 대화방을 나타냅니다. 스테이지 최소 정의 **원본 준비**, floor 위치 및 장치를 사용 하려는 위치 정방향 방향을 공간 좌표계에서 가운데에 사용자의 선택 합니다. Y=0 floor 평면에서이 단계 좌표계에서 콘텐츠를 배치 하 여 확인할 수 있습니다 프로그램 홀로그램 표시 편안 하 게 현장에서 사용자 준비 되 면 사용자에 게는 **고정 규모 환경**합니다.

### <a name="stage-bounds"></a>단계 경계

사용자가 정의할 수도 있습니다 **범위를 스테이징**, 영역 내에서 이동 하려는 위치는 가구 삭제 했으므로 이러한 대화방에서 혼합 현실입니다. 앱을 빌드할 수 있습니다 따라서 하는 경우는 **방 규모 환경**, 홀로그램 사용자 연결할 수 있는 배치 항상 됩니다 되도록 이러한 범위를 사용 하 여 합니다.

단계 참조 프레임 단일 floor 관련 콘텐츠를 배치 하는 좌표계를 고정 하면 되므로 이식 고정 크기 조정 및 가상 현실 헤드셋 용으로 개발 된 대화방 규모의 응용 프로그램에 대 한 가장 쉬운 경로입니다. 그러나 해당 VR 플랫폼을 사용 하 여 단일 좌표 시스템의 5 개 측정기 (16 피트) 지름에 대 한 콘텐츠를 안정화만 수, 하는 대로 레버 arm 전에 효과 사용 하면 콘텐츠 이동할 눈에 띄게 시스템을 조정 하는 대로 center 멀리 합니다. 5 미터를 넘어서 공간 앵커 필요 합니다.

## <a name="building-a-world-scale-experience"></a>전 세계 규모 환경 구축

HoloLens true 허용 **세계 규모 환경** 5m 초과 배회 하는 사용자를 수 있는 합니다. 전 세계 규모 앱을 작성 하는 대화방 규모 환경에 사용 되는 것 이상의 새로운 기술 해야 합니다.

### <a name="why-a-single-rigid-coordinate-system-cannot-be-used-beyond-5-meters"></a>왜 5m 초과 단일 고정 좌표계를 사용할 수 없습니다

오늘날 게임, 데이터 시각화 앱 또는 가상 현실 앱을 작성할 때 다른 모든 좌표가 다시 안정적으로 매핑할 수 있는 하나의 절대 세계 좌표 시스템을 설정 하는 일반적인 방법이입니다. 해당 환경에서 해당 환경에서 두 개체 간의 관계를 정의 하는 안정적인 변환을 찾을 수 있습니다. 이러한 개체를 이동 하지 않은 경우 해당 상대 변환은 항상 동일 합니다. 이 유형의 전역 좌표계를 미리 알고 있는 기 하 도형의 모든 순수 가상 세계를 렌더링 하는 경우에 작동 합니다. 대화방 규모 VR 앱 현재 일반적으로 설정 이러한 종류의 현장에서 해당 원본과 좌표계 절대 공간 크기 조정 합니다.

반면, HoloLens와 같은 제한 되지 않는 혼합된 현실 장치를는 건물의 층 전체에서 여러 미터를 안내는 대로 사용자의 환경 시간 경과 따른 정보를 지속적으로 조정 전 세계에는 동적 센서 기반 이해 합니다. 전 세계 규모 경험을 고정 된 단일 좌표계에 모든 사용자 제공을 배치 하면 전 세계를 기준으로 또는 서로 다른 시간이 지남에 따라 이러한 홀로그램 드리프트 반드시는 합니다.

예를 들어, 헤드셋 현재 4 미터 떨어진에 수 전 세계의 두 위치를 생각 하 고를 나중에 해당를 이해 하 고 위치는 3.9 미터 떨어진 실제로 학습을 구체화할 수 있습니다. 해당 홀로그램이 처음에 배치 되 면 4 미터는 단일 고정 좌표계에서 떨어져, 그 중 하나는 항상 표시 실제에서 해제 0.1 미터입니다.

### <a name="spatial-anchors"></a>공간 앵커

Windows Mixed Reality를 만들 수 있도록 하 여 이전 섹션에서 설명 하는 문제 해결 [공간 앵커](spatial-anchors.md) 전 세계 사용자가 제공을 배치 하는 위치에서 중요 한 요소를 표시 합니다. 공간 앵커를 추적 하는 시스템은 시간이 지남에 따라 세계는 중요 한 요소를 나타냅니다.

장치는 전 세계에 대 한 학습, 이러한 공간 앵커 각 앵커 실제 상대적으로 배치 된 해당 위치 정확 하 게 유지 됩니다 있도록 필요에 따라 해당 위치와 서로 상대적으로 조정할 수 있습니다. 여기서 사용자 배치를 홀로그램 위치의 공간 앵커를를 추가 하 고 다음 해당 홀로그램 공간 앵커에 상대적인 위치를 확인할 수 있습니다를 홀로그램 최적의 안정성을 유지 함을 미터 수십에서 로밍 사용자도.

공간 앵커와 서로 상대적으로이 연속 조정이 coordinate systems 앵커 공간에서 고정 참조 프레임 사이의 주요 차이점:

* 고정 참조 프레임 모든 배치 홀로그램 서로에 대 한 고정 관계를 유지 합니다. 그러나은 사용자가 멀리 떨어진 곳에 따라 해당 프레임의 좌표계 홀로그램 사용자 옆의 안정적인 나타나는지 확인을 기준으로 드리프트 수 있습니다.

* 또한 단계 참조 프레임에에서 배치 홀로그램 서로 고정 된 관계를 유지 합니다. 고정 프레임 달리 단계 프레임 항상 고정 계속 적용 됩니다는 정의 된 실제 원점 기준으로 합니다. 그러나 해당 5 미터 경계를 벗어나 스테이지의 좌표계에서 렌더링 되는 내용은 사용자가 해당 경계 내에서 준비 하는 동안만 안정적인 나타납니다.

* 공간 앵커 홀로그램 기준으로 드리프트 수 하나를 사용 하 여 배치 홀로그램 다른 공간 앵커를 사용 하 여 배치 합니다. 이 경우에, 예를 들어, 하나의 앵커 왼쪽 자동으로 조정 해야 하 고 다른 앵커를 오른쪽 조정 해야 합니다. 각 공간 앵커 위치 이해를 향상 시키는 Windows 수 있습니다.

고정 참조 프레임, 항상 사용자 근처 안정성에 대 한 최적화에 달리 단계 참조 프레임 및 공간 앵커 근처의 안정성을 확인 합니다. 이러한 홀로그램 시간이 지남에 따라 그대로 정확 하 게 유지 되지만 해당 좌표계 원점에서 너무 멀리 떨어져 렌더링 홀로그램 점점 더 심각한 레버 arm 효과 경험할를 의미 합니다. 왜냐하면 위치와 방향을 단계 또는 앵커를 미세 조정 확대 되는 앵커에서 거리에 비례하여입니다. 

경험에의 하면 해당 원본에 대 한 3 미터 이내 인지를 렌더링 하는 모든 먼 공간 앵커의 좌표계에 따라 확인 하는 것입니다. 주변 단계 원점의 먼 콘텐츠 렌더링은 확인을 증가 위치 오류는 사용자의 보기에서 가능한 한 많은 이동 하는 작은 홀로그램 영향이입니다.

### <a name="spatial-anchor-persistence"></a>공간 앵커 지 속성

공간 앵커에 앱 앱을 일시 중단 되거나 장치 종료 후에 중요 한 위치를 기억 하기를 사용할 수 있습니다.

앱을 만듭니다. 공간 앵커를 디스크에 저장 하 고 다음 로드할 수 이러한 다시 나중에 다시 유지 하 여 앱의 이들을 **앵커 저장소 공간**합니다. 저장 또는 앵커를 로드 하는 경우 나중에 앵커를 식별 하기 위해 앱에 의미 있는 문자열 키를 제공 합니다. 이 키에 앵커에 대 한 파일 이름으로 간주 합니다. 앵커를 사용 하 여 다른 데이터를 연결 하려는 경우 사용자는 해당 위치에 배치 하는 3D 모델, 앱의 로컬 저장소에 저장 하 고 선택한 키를 사용 하 여 연결 합니다.

사용자는 저장소에 유지 앵커 하 여 개별 홀로그램 배치 하거나 작업 영역을 중심점입니다 앱의 다양 한 홀로그램 배치 되며 나중에 이러한 홀로그램 찾을 위치 기대를 통해 앱의 다양 한 용도로 배치할 수 있습니다.

사용할 수도 있습니다 <a href="https://docs.microsoft.com/azure/spatial-anchors/overview" target="_blank">Azure 공간 앵커</a> HoloLens, iOS 및 Android 장치에서 비동기 홀로그램 지 속성에 대 한 합니다.  견고한 클라우드 공간 앵커를 공유 하 여 여러 장치 관찰할 수 있습니다 동일한 지속형된 홀로그램 시간이 지남에 따라 해당 장치를 동시에 함께 있는 경우에 합니다.

### <a name="spatial-anchor-sharing"></a>공간 앵커 공유

앱에서 실시간 공간 앵커를 공유할 수 있습니다 다른 장치를 사용 하 여 환경을 공유 실시간 허용 합니다.

사용 하 여 <a href="https://docs.microsoft.com/azure/spatial-anchors/overview" target="_blank">Azure 공간 앵커</a>, 여러 HoloLens, iOS 및 Android 장치에서 앱 공간 앵커를 공유할 수 있습니다. 동일한 공간 앵커를 사용 하 여 홀로그램을 렌더링 하는 각 장치를가지고 있으므로, 모든 사용자는 실제 환경에서 같은 위치에서 표시 홀로그램을 표시 됩니다.

## <a name="avoid-head-locked-content"></a>콘텐츠 헤드 잠긴 방지

강력한 렌더링 헤드 잠긴 콘텐츠 (예: HUD) 표시에서 고정된 한 지점을 유지 되는 것이 좋습니다. 일반적으로 콘텐츠 헤드 잠긴 사용자가 편리 하 게 되지 않으며 세계의 자연 스러운 일부 처럼 생각 하지 않습니다.

콘텐츠 헤드 잠긴 사용자에 게 연결 된 자체 세계에 배치 된 제공을 사용 하 여 일반적으로 대체 되어야 합니다. 예를 들어 [커서](cursors.md) 해야 일반적으로 푸시할 수 전 세계에 사용자의 게이즈 아래에 있는 개체의 거리와 위치를 반영 하도록 자연스럽 게 확장 합니다.

## <a name="handling-tracking-errors"></a>추적 오류 처리

어두운 복도 등 일부 환경에서는 내부 스케일 아웃 추적을 사용 하 여 자체 세계에서 올바르게 찾을 헤드셋에 대 한 수 없습니다. 이 홀로그램 표시 되지 않습니다 또는 잘못 처리 하는 경우 잘못 된 위치에 표시 될 수 있습니다. 이제는 다음이 발생할 수 있습니다, 조건을 사용자 경험에 미치는 영향을 설명 하 고 최선의 팁이이 상황을 처리 합니다.

### <a name="headset-cannot-track-due-to-insufficient-sensor-data"></a>헤드셋 부족 하 여 센서 데이터 인해 추적할 수 없습니다.

헤드셋의 센서 헤드셋 인 알 수 없는 경우도 있습니다. 이 대화방 꺼져 십자선 또는 손, 센서 나와 하거나 충분 한 텍스처는 주변에 없는 경우 발생할 수 있습니다.

이 경우 헤드셋 world 잠긴 홀로그램 렌더링에 충분 한 정확도 사용 하 여 해당 위치를 추적할 수 없습니다. 여기서는 공간 앵커, 고정 프레임 또는 스테이지 프레임은 기준으로 장치를 알아낼 수 없지만 여전히 연결 된 프레임의 참조에서 본문 잠긴 콘텐츠를 렌더링할 수 있습니다.

앱 사용자는 센서를 발견할 자세한 표시등 켜기 등의 몇 가지 팁을 설명 하는 몇 가지 대체 (fallback) 본문 잠긴 콘텐츠를 렌더링 다시 추적 위치 하는 방법을 알려주기 해야 합니다.

### <a name="headset-tracks-incorrectly-due-to-dynamic-changes-in-the-environment"></a>환경에서 동적 변경으로 인해 잘못 헤드셋 추적

경우에 따라 많은 방에 따라 이동 하는 많은 사람들이 같은 환경에서 동적 변경 사항이 있는 경우 장치가 제대로 추적할 수 없습니다. 이 경우에 홀로그램 이동이 동적 환경에서 자체를 추적 하는 장치를 시도할 때 드리프트를 보일 수 있습니다. 이 시나리오에 도달한 경우 보다 동적 환경에서 장치를 사용 하는 것이 좋습니다.

### <a name="headset-tracks-incorrectly-because-the-environment-has-changed-significantly-over-time"></a>헤드셋 추적 하지 올바르게 환경 시간이 지남에 따라 상당히 변경 되었기 때문에

경우에 따라 시작할 때 많은 변경이 이뤄졌습니다는 환경에서 헤드셋을 사용 하 여 (중요 한 이동을 예를 들어 가구 벽 hangings 등), 수 원래 위치에서 일부 홀로그램 만큼 나타날 수 있습니다. 사용자가이 새 공간에서 이동 하는 대로 이전 홀로그램 주위 이동할 수도 있습니다. 시스템의 이해 공간 이상 보유 하 고 환경 대화방의 기능을 조정 하는 동안 매핑할 시도 때문입니다. 이 시나리오에서는 것이 좋습니다. 다시 배치 홀로그램 위치 나타나지 않는 경우 전 세계에 고정 하는 것을 권장 합니다.

### <a name="headset-tracks-incorrectly-due-to-identical-spaces-in-an-environment"></a>환경에서 동일한 공간으로 인해 잘못 헤드셋 추적

경우에 따라 홈 또는 다른 공간에서 두 개의 동일한 영역 있을 수 있습니다. 예를 들어 두 개의 동일한 회의실, 동일한 두 영역에서 장치의 시야를 포함 하는 두 개의 대형 동일한 포스터 모서리입니다. 이러한 시나리오에서는 장치는 동일한 구성 요소 간에 혼동 하며 내부 표현에 있는 동일한으로 표시할 때때로 수 있습니다. 이 홀로그램 다른 위치에 표시 하려면 몇 가지 영역에서 않을 수 있습니다. 장치는 환경의 내부 표현 손상 되었기 때문에 종종 추적 손실 시작할 수 있습니다. 이 경우 것이 좋습니다. 시스템의 환경 이해를 다시 설정 합니다. 모든 공간 앵커 배치 손실로 이어집니다 map를 다시 설정 하면 note 하십시오. 그러면 헤드셋 환경의 고유한 영역에서 잘 추적할 수 있습니다. 그러나 동일한 영역에 다시 사이 장치 가져옵니다 혼동 하는 경우에 다시 문제가 발생할 수 있습니다.
