훌륭한 Windows Mixed Reality 환경을 제공 하기 위해 노력 했습니다 있지만 몇 가지 알려진된 문제가 여전히 추적 하는 것입니다. 다른 사용자가 있다면 하세요 [의견](give-us-feedback.md)합니다.

### <a name="hololens"></a>HoloLens

#### <a name="after-update"></a>업데이트 후

에 HoloLens에 RS4로 RS1에서 업데이트 한 후 다음과 같은 문제가 나타날 수 있습니다.
* **Pin 재설정** -업데이트 후 시작 메뉴에 앱 고정 3x3를 기본값으로 재설정 됩니다. 
* **앱 및 배치 홀로그램 재설정** -사용자가 속한 환경에 배치 하는 앱 업데이트 후 제거 되 고 공간에 전체 다시 배치 해야 합니다. 
* **피드백 허브 즉시 시작 되지 않는** -즉시 업데이트 후 자체적으로 업데이트 하는 동안 피드백 허브와 같은 일부 수신함 앱 시작 수까지 몇 분 정도 걸립니다. 
* **회사 Wi-fi 인증서 다시 동기화 해야** -는 HoloLens 회사 인증서를 다시 연결할 수 있게 되기 전에 장치에 다시 동기화를 위해 다른 네트워크에 연결 해야 하는 문제를 조사 하는 것 인증서를 사용 하 여 회사 네트워크입니다. 
* **아닌 H.265 HEVC 비디오 재생 되지 않으면** -아닌 H.265 비디오를 재생 하려고 하는 응용 프로그램에 오류 메시지가 표시 됩니다. 문제를 해결 하는 것 [Windows Device Portal 액세스](using-the-windows-device-portal.md)를 선택 **앱** 왼쪽된 탐색 모음에서와 **제거** HEVC 응용 프로그램입니다. 그런 다음 최신 [HEVC Video Extension](https://www.microsoft.com/p/hevc-video-extensions/9nmzlz57r3t7) Microsoft Store. 문제를 조사 하는 것입니다. 

#### <a name="for-developers-updating-hololens-apps-for-devices-running-windows-10-april-2018-update"></a>개발자: Windows를 실행 한 장치용 앱 HoloLens 10 2018 년 4 월 업데이트

훌륭한 목록과 함께 [새로운 기능](#new-features-for-hololens), Windows 10 년 4 월 이전의 그렇지 않은 몇 가지 코드 동작을 적용 하는 HoloLens 2018 업데이트 (RS4):
* **중요 한 리소스 (카메라, 마이크 등)를 사용 하도록 사용 권한 요청**  -HoloLens에 RS4 카메라 또는 마이크와 같은 중요 한 리소스에 액세스 하려는 앱에 대 한 사용 권한 요청 적용 됩니다. HoloLens에 RS1 이러한 프롬프트를 강제로 적용 하지 않은, 앱, 이러한 리소스에 즉시 액세스의 경우이 충돌이 발생할 수 있습니다 RS4에서 (경우에 사용자가 요청된 된 리소스에 대 한 권한 부여). 관련를 읽어보세요 [UWP 앱 기능 선언 문서](https://docs.microsoft.com/windows/uwp/packaging/app-capability-declarations) 자세한 내용은 합니다.
* **외부 사용자 고유의 앱에 대 한 호출** -RS4 HoloLens에 적절 한 사용에 적용 됩니다는 [ *Windows.System.Launcher* 클래스](https://docs.microsoft.com/uwp/api/Windows.System.Launcher) 사용자 자신의 서로 다른 앱을 시작 합니다. 예를 들어 살펴본 문제를 호출 하는 앱을 사용 하 여 *Windows.System.Launcher.LaunchUriForResultsAsync* ASTA 비 UI 스레드에서 합니다. HoloLens에서 RS1에 성공이 있지만 RS4 UI 스레드에서 실행할 수에 대 한 호출을 해야 합니다.

#### <a name="audio"></a>오디오

* Windows Mixed Reality 헤드셋을 사용 하기 전에 호스트 PC에서 사용 하도록 설정 하는 Cortana가 Windows Mixed Reality 홈 주위 배치 앱에 적용 되는 공간 사운드 시뮬레이션을 손실 될 수 있습니다. 
   * 해결 방법은 PC도 헤드셋에 연결 된 오디오 장치에 연결 된 모든 오디오 장치에서 "Windows Sonic에 대 한 헤드폰"를 사용 하도록 설정 하려면 것:
      1. 바탕 화면 작업 표시줄에서 화자 아이콘을 마우스 왼쪽 단추로 클릭 하 고 오디오 장치 목록에서 선택 합니다.
      2. 바탕 화면 작업 표시줄에서 화자 아이콘을 마우스 오른쪽 단추로 클릭 하 고 "스피커 setup" 메뉴에서 "Windows Sonic에 대 한 헤드폰"를 선택 합니다.
      3. 모든 오디오 장치 (끝점)에 대해 이러한 단계를 반복 합니다.
   * 또 다른 옵션은 "Let Cortana 대응 Hey Cortana" 해제 **설정을** > **Cortana** Windows Mixed Reality를 시작 하기 전에 바탕 화면에 있습니다.
* Windows Mixed Reality 헤드셋와 같은 USB 허브 (외부 또는 내부 PC)를 공유 하는 다른 멀티미디어 USB 장치 (예: 웹 캠), 드문 경우에서 헤드셋의 오디오 잭이/헤드폰 있을 수 있습니다 하거나 버 징 사운드 또는 오디오 없는 모든. 다른 장치와 같은 허브를 공유 하거나 하지 다른 USB 멀티미디어 장치 연결 끊기/해제 하는 USB 포트에 헤드셋 하 여 해결할 수 있습니다.
* 매우 드문 경우에는 호스트 PC의 USB 허브 Windows Mixed Reality 헤드셋 데 필요한 기능 부족을 제공할 수 없습니다 및 헤드셋 연결할 헤드폰 방해 폭주 표시 될 수도 있습니다.

#### <a name="holograms"></a>홀로그램

* 홈에 Windows Mixed Reality에 많은 수의 홀로그램 배치한, 경우 일부 수 사라지고 분야를 살펴보면 대로 다시 나타납니다. 이 방지 하려면 해당 영역의 홈 Windows Mixed Reality를 홀로그램 중 일부를 제거 합니다.

#### <a name="motion-controllers"></a>컨트롤러 동작

* 입력 헤드셋에 라우팅되지 않습니다, 경우 동작 컨트롤러 방 경계 옆에 있는 대기 중인 경우 간단 하 게 사라집니다 수 있습니다. Win + Y 파란색 배너가 데스크톱 모니터에서이 키를 눌러이 확인 됩니다. 
* 경우에 따라 Microsoft Edge에서 웹 페이지를 클릭 하면 콘텐츠를 클릭 하는 대신 확대 됩니다.

#### <a name="desktop-app-in-the-windows-mixed-reality-home"></a>Windows Mixed Reality 홈에서 데스크톱 앱

* 캡처 도구는 데스크톱 앱에서 작동 하지 않습니다.
* 데스크톱 앱에 다시 시작할 때 설정이 유지 되지 않습니다.
* 혼합 현실 포털 미리 보기를 사용 하 여 Windows Mixed Reality 홈에서 데스크톱 앱을 열 때 바탕 화면에, 무한 미러 효과 확인할 수 있습니다. 
* 데스크톱 앱을 실행 문제를 일으킬 수 성능에 비-Ultra Windows 혼합 현실 Pc; 권장 되지 않습니다.  
* 데스크톱 앱 수 자동으로 시작은 보이지 않는 창을 바탕 화면에서 포커스를가지고 때문입니다. 
* 데스크톱 사용자 계정 컨트롤 프롬프트 헤드셋 프롬프트 완료 될 때까지 검정색을 표시 하 게 합니다.

#### <a name="windows-mixed-reality-for-steamvr"></a>Windows Mixed Reality SteamVR에 대 한

* 혼합 현실 포털을 시작 해야 할 수 있습니다를 업데이트 한 후에 보장 하기 위해 필요한 소프트웨어 업데이트는 Windows 10 2018 년 4 월 업데이트 전에 끝내 야 SteamVR를 시작 합니다. 
* 최신 버전의 Windows를 사용 하 여 호환성을 유지 해야 SteamVR에 대 한 Windows Mixed Reality에 있어야 합니다. 10 2018 년 4 월 업데이트 합니다. 자동 업데이트에 대해 켜 집니다 Windows Mixed Reality Steam에서 라이브러리의 "소프트웨어" 섹션에 있는 SteamVR에 있는지 확인 합니다.  

#### <a name="other-issues"></a>기타 문제

>[!IMPORTANT]
>초기 버전의 Windows 10 2018 년 4 월 업데이트 (버전 17134.5) 참가자 푸시 Windows Mixed Reality를 실행 하는 데 필요한 소프트웨어가 없습니다. Windows Mixed Reality를 사용 하는 경우이 버전을 방지 하는 것이 좋습니다. 

초기 릴리스의 향후 업데이트 패치를 수정 하기 위해 노력 하는이 업데이트 (10.0.17134.1)에서 Surface Book 2를 사용 하는 경우 성능 저하가 확인 되었습니다. 수동으로 업데이트 하거나 일반적으로 롤아웃 하려면 업데이트에 대 한 대기 하기 전에 해결이 될 때까지 대기 하는 것이 좋습니다.
