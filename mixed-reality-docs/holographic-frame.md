# <a name="holographic-frame"></a>Holographic 프레임

사용자가 해당 헤드셋에서 제공 하는 사각형 뷰포트 통해 혼합된 현실 세계를 참조 하세요. HoloLens,이 사각형 영역 holographic 프레임 라고에 이들을 실제에 오버레이된 디지털 콘텐츠를 볼 수 있습니다. Holographic 프레임에 대 한 액세스에 최적화 된 환경을 디자인 기회, 문제를 완화 만들고 혼합된 현실 응용 프로그램의 사용자 환경을 개선 합니다.

## <a name="designing-for-content"></a>콘텐츠에 대 한 디자인

종종 디자이너가 필요 하다 고 무엇이 사용자 즉시 표시를 사용자 환경의 범위를 제한 하려면, 전체에서 개체를 볼 사용자 확인 실제 크기를 희생 합니다. 마찬가지로 복잡 한 응용 프로그램을 사용 하 여 디자이너 종종 오버 로드 콘텐츠로 holographic 프레임 어려운 상호 작용 및 혼잡된 인터페이스를 사용 하 여 사용자가 부담 합니다. 혼합된 현실 콘텐츠 만들기 디자이너 필요한 경험을 하 여 즉시 해당 뷰에서 사용자 앞에 직접을 제한 하지 않습니다. 실제 사용자 주위에 매핑된 경우 그런 다음 이러한 모든 표면 고려해 야 디지털 콘텐츠 및 상호 작용에 대 한 잠재적인 캔버스입니다. 상호 작용 및 콘텐츠는 환경 내에서 적절 한 설계 해당 공간, 해당 주의 주요 콘텐츠를 전송 및 혼합된 현실의 잠재력을 참조 하는 데 도움이 이동할 사용자를 권장 해야 합니다.

가장 중요 한 기술을 하도록 이동 하는 앱 내에서 탐색을 가장 하 여 **사용자가 환경을 조정할 수 있도록**입니다. 장치를 사용 하 여 짧은 기간 ' 작업-무료 ' 사용자에 게 제공 합니다. 이 공간에 개체를 배치 하 고 사용자가 이동할 수 있도록 또는 환경에 대 한 소개를 설명 처럼 간단할 수 있습니다. 특정 제스처 (예: air-누르기), 대신 사용자 상호 작용을 요구 하거나 앱의 단계를 통해 진행 하기 전에 장치를 통해 콘텐츠를 보기에 수용할 수 있도록 목적 또는이 이번 모든 중요 한 작업의 무료 여야 합니다. 이 사용자의 장치를 사용 하 여 처음으로이 경우 holographic 프레임 및 홀로그램의 특성을 통해 보는 편리 하 게 콘텐츠를 가져오기는 특히 중요 합니다.

### <a name="large-objects"></a>큰 개체

종종 환경을 호출에 대 한 콘텐츠를 특히 실제 콘텐츠를 보다 커지는 경우가 holographic 프레임입니다. (더 작은 규모로 또는 거리에서)에 처음 도입 된은 경우에 맞게 holographic 프레임 내에서 일반적으로 표시할 수 없는 개체를 축소할 수 해야 합니다. 핵심은 **개체의 전체 크기를 확인할 수 있도록** 전에 확장 없게 프레임을 만듭니다. 예를 들어, holographic 코끼리 표시할지를 크기 조정 하기 전에 공간 이해가 동물의 전체적인 모양 형성 하도록 허용 하는 프레임 내에서 완벽 하 게 맞게 [실제 확장](scale.md) 사용자 근처 합니다.

염두에서 개체의 전체 크기를 사용 하 여 사용자는 다음 이동 하 고 해당 개체의 특정 부분을 검색할 수 있는 위치를 기대를 갖습니다. 마찬가지로, 몰입 형 콘텐츠를 사용 하 여 환경의 도움이 해당 콘텐츠의 전체 크기를 다시 참조 하기 위해 몇 가지 방법이 있습니다. 예를 들어 환경을 포함 가상 집 모델 탐색을 하는 경우 사용자가 집 내는 것을 이해 하려면 트리거할 수 있는 환경의 작은 doll 집 크기 버전이 도움이 될 수입니다.

큰 개체에 대 한 디자인의 예제를 참조 하세요 [Volvo 자동차](holographic-frame.md#volvo-cars)합니다.

### <a name="many-objects"></a>개체의 수

여러 개체 또는 구성 요소를 사용 하 여 환경을 사용자 주위의 전체 공간을 사용 하 여 사용자 앞에 직접 holographic 프레임을 어지럽히지 않기 위해 고려해 야 합니다. 일반적으로 느린는 환경에 콘텐츠를 도입 하는 것이 좋습니다 및 사용자에 게 많은 개체를 제공 하도록 계획 하는 환경을 사용 하 여 특히 그렇습니다. 큰 개체를 사용 하 여 핵심 하는 것과 마찬가지로 **콘텐츠의 레이아웃을 이해 하는 사용자가** 환경에서 돕고 콘텐츠 환경에 추가 됨에 따라 이들을 이란 공간 이해 합니다.

이 작업을 수행 하는 기술 앵커 콘텐츠를 실제 환경에서 영구 요소 (랜드마크가 라고도 함)를 제공 하는 것입니다. 예를 들어, 한 랜드마크 실제, 디지털 콘텐츠 표시 되는 테이블 또는 콘텐츠 자주 표시 되는 디지털 화면 집합과 같은 디지털 개체를 같은에서 실제 개체를 수 있습니다. 개체를 주변 초과 콘텐츠의 검색으로 도움이 될 수 있지만 키 콘텐츠를 찾습니다는 사용자를 장려 하기 위해 holographic 프레임의 주변에도 배치할 수 있습니다 [주의 이사회](holographic-frame.md#attention-directors)합니다.

검색할 사용자를 유도할 수 있는 주변에 개체를 배치 및 아래 설명 된 대로 주의 directors를으로 도움이 될이 합니다.

## <a name="user-comfort"></a>사용자 편안 함

큰 개체 또는 여러 개체를 사용 하 여 혼합된 현실 환경에 대 한 것이 얼마나 많은 헤드를 고려해 야 할 중요 한 및 콘텐츠와 상호 작용 하기 목 이동이 필요 합니다. 환경은 헤드 이동 측면에서 세 가지 범주로 나눌 수 있습니다. **가로** (좌우로), **세로** (아래로), 또는 **몰입 형** (가로 및 세로). 가능한 경우 제한 상호 작용을 가로 또는 세로 범주 대부분이 이상적 중립 위치에는 사용자의 헤드 holographic 프레임의 가운데에 수행 하는 대부분의 환경으로 합니다. 지속적으로 해당 보기에는 비 자연 헤드 위치 (예: 항상 키 메뉴 상호 작용을 액세스 하려면 조회)을 이동할 대상 사용자는 상호 작용을 하지 않습니다.

가로 헤드 이동 더 [편리 하 게](comfort.md) 일반적이 지 않은 이벤트에 대 한 세로 이동 예약 되어야 하는 동안 자주 상호 작용 합니다. 예를 들어, 긴 가로 시간 표시 막대를 포함 하는 환경 (예: 메뉴 확인) 상호 작용에 대 한 세로 헤드 이동을 제한 해야 합니다.

사용자의 공간 관련 개체를 배치 하 여 방금 헤드 이동 하지 않고 전체 본문 움직임을 장려 하는 것이 좋습니다. 개체 또는 큰 개체를 이동 하는 환경을 가로 및 세로 축 따라 자주 이동 필요한 특히 헤드 이동 하려면 특별 한 주의 해야 합니다.

## <a name="interaction-considerations"></a>상호 작용 고려 사항

콘텐츠를에서와 마찬가지로 혼합된 현실 환경에서 상호 작용 해야 사용자 수 즉시으로 표시 되는 내용으로 제한 수 없습니다. 상호 작용에서에서 발생할 수 있습니다 어디서 나 사용자 주위의 실제 공간 및 이러한 상호 작용 있습니다 사용자 환경을 탐색 하 고 이동 하는 것이 좋습니다.

### <a name="attention-directors"></a>주의 이사

주요 상호 작용 나 관심 지점을 나타내는 환경을 통해 작업을 진행 사용자에 게 중요 한 수 있습니다. 사용자 주 및 holographic 프레임의 이동 어렵거나 투박한 방법으로 리디렉션될 수 있습니다. 사용자 과부하를 피하기 위해 (특히 환경을 시작)에서 혼합된 현실에서 사용 가능한 탐색 기간을 사용 하 여 주의 이사회 균형을 유지 해야 합니다. 일반적으로 두 가지가 주의 이사회:
* **Visual 감독:** 사용자에 게 알리는 특정 방향으로 이동 해야 하는 가장 쉬운 방법은 시각적 표시를 제공 하는 것입니다. 시각 효과 (예: 사용자 경험의 다음 부분으로 시각적으로 따르면 경로)를 통해 나이 수행할 수 있습니다 간단한 방향 화살표도 합니다. '에 연결 되어 있지 ' holographic 프레임 또는 커서 사용자의 환경 내에서 모든 시각적 표시기 해야 하는 참고 접지 합니다.
* **오디오 감독:** [공간 소리](spatial-sound-design.md) (사용자 환경에 들어가는 개체의 경고)은 장면에 개체를 설정 또는 (키 개체에 대 한 사용자의 보기 이동) 하는 공간에서 특정 시점에 주목해 강력한 방법을 제공할 수 있습니다. 오디오 이사를 사용 하 여 사용자의 주의 안내 하는 보다 미묘 하 고 visual 이사회 보다 개입 수준이 낮습니다 수 있습니다. 경우에 따라 최상의 오디오 director를 사용 하 여 시작 하 고 사용자는 큐를 인식 하지 못하면 visual 디렉터도 이동할 수 있습니다. 오디오 이사회 추가 강조를 위해 visual 이사와도 연결할 수 있습니다.

### <a name="commanding-navigation-and-menus"></a>명령, 탐색 및 메뉴

혼합된 현실 환경의 인터페이스 이상적으로 쌍을 이루는 긴밀 하 게 제어 하는 디지털 콘텐츠입니다. 따라서 부동성 2D 메뉴 상호 작용에 적합 하지 않은 경우가 많습니다 및 holographic 프레임 내에서 사용 하 여 편리 하 게 사용자가 어려울 수 있습니다. 메뉴 또는 텍스트 필드와 같은 인터페이스 요소 필요가 없는 환경을 사용 하 여는 것이 좋습니다는 [tag-along 메서드](billboarding-and-tag-along.md) 짧은 지연 후 holographic 프레임을 따라야 합니다. 잠금을 사용 하지 않는 헤드업 디스플레이 처럼 프레임에 내용으로이 사용자 및 중단 기타 디지털 장면의 개체에 대 한 집중 교육의 의미에 대 한 울 수 있습니다.

수도 인터페이스 요소를 특정 직접 배치 콘텐츠 제어 하는, 사용자의 물리적 공간 주위 자연스럽 게 발생 하는 상호 작용을 허용 합니다. 예를 들어, 별도 파트로 복잡 한 메뉴를 중단 합니다. 각 단추나 그룹 컨트롤을 사용 하 여 상호 작용에 영향을 특정 개체에 연결 합니다. 이 개념을 더 이상 사용 하려면 사용을 고려 [상호 작용할 수 없는 개체](interactable-object.md)합니다.

### <a name="gaze-and-gaze-targeting"></a>응시 및 게이즈 대상 지정

Holographic 프레임 트리거 상호 작용에 개발자를 위한 도구를 제공 뿐만 아니라 사용자의 주의 주의 평가 합니다. [Gaze](gaze.md) 중 하나인 합니다 [HoloLens에 상호 작용 키](interaction-fundamentals.md)을 응시와 연결할 수 있습니다 [제스처](gestures.md) (같은 어 탭을 사용 하 여) 또는 [음성](voice-input.md) (허용에 대 한 짧은 를 좀 더 자연 스러운 음성 기반 상호 작용). 따라서 이렇게 하면 두 holographic 프레임 디지털 콘텐츠 관찰 수 있을 뿐만 아니라와 상호 작용에 대 한 공간. 환경을 사용자 공간 (예를 들어 게이즈 + 제스처를 사용 하 여 사용자의 공간 주위 다중 선택 개체) 관련 여러 개체와 상호 작용에 대 한 호출을 하는 경우 필요한 헤드의 양을 제한 하거나 사용자의 보기에 해당 개체를 가져오는 것이 좋습니다 승격로 이동을 [사용자와 쾌적](comfort.md)합니다.

게이즈 환경을 통해 사용자의 주의 추적 하 고 개체 참조를 사용할 수도 있습니다 또는 장면 사용자 부분에 가장 주의를 지불 합니다. 사용자가 가장 많이 소비 한다면 보려는 열 지도 시간 또는 특정 개체 또는 상호 작용 하는 같은 분석 도구에 대 한 허용 된 경험을 디버깅을 위해 사용 하 여 특히 수 있습니다. 추적 게이즈 환경에서 매체에 대 한 강력한 도구를 제공할 수도 있습니다 (참조를 [Lowe의 주방](holographic-frame.md#lowes-kitchen) 예제).

## <a name="performance"></a>성능

Holographic 프레임의 적절 한 사용은 기본 합니다 [성능 품질](understanding-performance-for-mixed-reality.md) 경험 합니다. 일반적인 기술 (및 유용성) 문제에는 디지털 콘텐츠를 렌더링 성능이 저하를 일으키는 사용 하 여 사용자의 프레임 오버 로드 됩니다. 고려 렌더링의 부담을 줄일 수는 최적의 디스플레이 품질 보장을 위에서 설명한 기술을 사용 하 여 대신 사용자 주위의 전체 공간을 사용 하 여 디지털 콘텐츠를 정렬 하를 합니다.

HoloLens의 holographic 프레임 내에서 디지털 콘텐츠와 연결할 수도 있습니다는 [안정화 평면](case-study-using-the-stabilization-plane-to-reduce-holographic-turbulence.md) 최적의 성능을 위해 및 [홀로그램 안정성](hologram-stability.md)합니다.

Volvo 직면해 holographic 프레임: 큰 자동차 사용자 옆에 있는 오른쪽 입력에 너무 큽니다. 솔루션을 실제 랜드마크 테이블 위에 배치 하는 자동차 작은 디지털 모델로 전시장의 중앙 테이블을 사용 하 여 환경을 시작 하는 것 이었습니다. 이렇게 하면 사용자가 일어나는 전체 자동차 것을 도입할 때 자동차 환경의 나중에 해당 실제 크기 증가 되 면 공간 이해의 의미에 대 한 허용 합니다.

Volvo의 환경을 사용 하면 또한 소규모 자동차 모델 테이블에에서 표시 방에 벽 장기 시각 효과 만드는 visual directors를 사용 합니다. 따라서 '매직 창' 효과 실제 대규모로 자동차의 추가 기능을 보여 주는, 거리에 자동차 전체 보기를 표시 합니다. 헤드 이동은 가로 (대신 수집 신호 시각적으로 및 환경의 Volvo 연결의 설명에서) 사용자에서 직접 상호 작용 없이 합니다.

### <a name="lowes-kitchen"></a>Lowe의 주방

Lowe의 한 저장소 환경에는 HoloLens를 통해 볼 수 있듯이 수리 중 다양 한 기회를 보여 주기 위해 주방의 두는 전면적인 mockup 고객을 초대 합니다. 저장소에 주방 어플라이언스와 싱크대에 펼치려면 혼합된 현실 환경에 대 한 캐비닛의 빈 캔버스가 디지털 개체에 대 한 실제 배경을 제공 합니다.

실제 화면 환경에서 자체를 포함 하는 사용자에 대 한 정적 랜드마크로 작용 프록시로 Lowe의 연결 다른 제품 옵션 및 완료 되는 사용자를 안내 합니다. 따라서에서 연관 '냉장고' 또는 '원의 주방' 쇼케이스 디지털 콘텐츠를 사용자의 주의 능력이 보낼 수 있습니다.

![Lowe 연결 HoloLens 환경을 통해 가이드 고객에 게 태블릿을 사용합니다.](images/loweskitchen-750px.jpg)<br>
*Lowe 연결 HoloLens 환경을 통해 가이드 고객에 게 태블릿을 사용합니다.*

부분적으로 사용자의 환경은 Lowe의 연결에 의해 제어 되는 태블릿 환경으로 관리 됩니다. 연결의 역할에 속하지 여기서 수도 주방에서 관심 지점 간에 원활 하 게 주의 전달 하는 과도 한 헤드 이동 제한할 수 있습니다. 또한 태블릿 경험 있는 사용자는 (예를 들어 생각에 잠기거나 그럴 cabinetry의 특정 영역)을 보다 정확 하 게 리 지침 제공을 이해 하는 데 도움이 주방 열 지도 보기의 형태로 게이즈 데이터로 Lowe의 연결을 제공 합니다.

Lowe의 주방 환경 보다 자세히 살펴볼을 참조 하세요 [Ignite 2016에서 Microsoft의 키 노트](https://www.youtube.com/watch?v=gC_4JxF0e_k)합니다.

### <a name="fragments"></a>조각

HoloLens 게임 조각에 있습니다 거실에 앉고에 앉아서 담 벼 락에 의존 하는 문자를 사용 하 여 문의할 가상의 모임 장소, 뿐만 아니라 단서 및 증명 정보를 보여 주는 가상 crime 장면으로 변환 됩니다.

![조각은 사용자의 홈 화면 및 실제 개체와 상호 작용 하는 문자를 사용 하 여 수행 하도록 설계 되었습니다.](images/fragments-750px.jpg)<br>
*조각은 사용자의 홈 화면 및 실제 개체와 상호 작용 하는 문자를 사용 하 여 수행 하도록 설계 되었습니다.*

사용자 경험을 처음 시작 하는 경우 이러한는 짧은 기간 조정, 거의 상호 작용이 필요한 지정 된 대신 하도록 권장 하 살펴보겠습니다. 그러면 대화방 제대로 게임의 대화형 콘텐츠에 대 한 매핑 확인 합니다.

환경 전체에서 문자는 초점 해지고 visual 이사회 (헤드 이동 자 찾거나 제스처 관심 영역을 포함 하도록 설정)로 작동 합니다. 또한 게임 사용자 개체 또는 이벤트를 찾는 데 너무 오랜 시간이 걸리는 경우 띄 시각 신호에 의존 하 고 특히 (장면 입력할 때 문자 음성)를 사용 하 여 공간 오디오를 많이 사용 합니다.

![가상 이야기로 떠 들 석 Aldrin 대상의 사용자는 중점이 됩니다. Mars 합니다.](images/destinationmars-750px.png)<br>
*가상 이야기로 떠 들 석 Aldrin 대상의 사용자는 중점이 됩니다. Mars 합니다.*

몰입 형 환경을으로 이러한 사용자 된 것이 좋습니다, 검색할 해당 헤드 가상 화성 가로 보려는 모든 방향으로 이동 합니다. 하지만 보장 하기 위해 사용자 편안 하 게 이야기로 떠 들 석 Aldrin 내레이션을 가상 환경 전체에서 초점을 제공 합니다. 이 가상 소식 기록 (만든 [Microsoft의 혼합 현실 캡처 Studios](https://www.microsoft.com/mixed-reality/capture-studios)) 완벽에 가까운 뷰에서 만날 수 있도록 대화방의 모퉁이에 있는 실제, 인적 자원 관리 크기로 구현. 소식의 설명을 전달 환경에서 다양 한 요점을 강조 하는 사용자 (예: 화성 집합이 거리에서 mountain 범위나 바닥에가 최고인) 특정 바뀌어 또는 그에 도입 된 개체를 사용 하 여 합니다.

![가상 narrators 환경 전체에서 강력한 초점을 만드는 사용자의 이동을 수행 기능을 합니다.](images/gazereset-750px.png)<br>
*가상 narrators 환경 전체에서 강력한 초점을 만드는 사용자의 이동을 수행 기능을 합니다.*

현실적인 표현의 소식 강력한 초점을 담당 하 고,을 말하는 것 처럼 느낄 사용자 쪽으로 대화를 설정 하기 위해 미묘한 기술을 전체를 제공 합니다. 사용자 환경에 대해 이동, 소식 전환 됩니다 앞쪽 임계값 사용자 자신의 주변 멀리 벗어나는 경우 중립적인 상태로 돌아가기 전에 합니다. 사용자 대화에서 완전히 (예를 들어 장면에서 다른 곳에서 살펴볼)으로 표시 되 면 다시 소식, 내레이터의 방향 위치는 한 번에 다시 초점을 맞춰야 사용자입니다. 다음과 같은 기술을 집중 교육의 강력한 의미를 제공 하 고 holographic 프레임 내에서 과도 한 헤드 이동 줄이고 승격 초점은 만듭니다 [사용자와 쾌적](comfort.md)합니다.
