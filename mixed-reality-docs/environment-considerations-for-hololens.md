# <a name="environment-considerations-for-hololens"></a>HoloLens에 대 한 환경 고려 사항

HoloLens 혼합 holographic "실제" 상황에서는 사용 하 여 홀로그램 환경에 배치 합니다. 벽에 "중단" holographic 앱 창, holographic ballerina는 담근에서 회전, 토끼 돌릴 자신도 모르는 친구의 head 위에 표시 합니다. 환경에 맞게 holographic 전 세계 분산 됩니다는 몰입 형 게임 또는 앱을 사용 하는 경우-하지만 여전히을 확인 하는 공간을 이동할 수 있게 됩니다.

장치를 해제 하는 경우에 여기서 배치한, 배치 홀로그램이 유지 됩니다. 

## <a name="setting-up-an-environment"></a>환경 설정

HoloLens 장치에 배치 하 여 안정적이 고 정확 하 게 제공 하는 방법을 알고 *추적* 공간에는 사용자입니다. 적절 한 추적 하지 않고 장치 인식 하지 못합니다 그-사용자나 환경 홀로그램 잘못 된 위치에 표시, 동일한 위치에 표시 된 되지 매번 하거나, 전혀 표시 되지 않을 수 있습니다.

성능 추적을 크게 받습니다.에서는 사용자가 환경에서 및는 과학 보다는 정교한 작업은 안정적이 고 일관 된 추적을 유도 하는 환경을 조정 합니다. 여러 다양 한 환경 요소는 함께 결합 하는 추적을 사용 하도록 설정 하지만 혼합 현실 개발자는 일부의 요인을 더 나은 추적을 위한 공간을 조정 하기 위해 반드시에서 유지할 수 있습니다.
 
### <a name="lighting"></a>조명
Windows Mixed Reality visual light를 사용 하 여 사용자의 위치를 추적 합니다. Environment 너무 되 면이 카메라를 포화 가져올 수 있습니다, 그리고 및 아무 것도 표시 됩니다. 환경 너무 어두운 색 이면 충분 한 정보를 카메라 선택할 수 없습니다 하 고 아무 것도 표시 됩니다. 조명 하 고 충분히 하더라도 밝은 사람이 볼 수 있는지 노력 없이 있지만 있으므로 밝은 하지 표시등이 살펴볼 번거로운 수 있어야 합니다.

영역 있는 전체 dim 영역에서 밝은 빛의 포인트도 문제가 카메라에 밝은 공간의 입출력 이동 하는 경우 조정 합니다. 이 인해 "손실" 가져오고 인지 light의 변경 위치를 변경 하는 장치가 있습니다. 더 나은 추적 영역에서 안정적인 램프 레벨을 일으킵니다.

태양 시간이 지남에 따라 크게 달라질 수 있으므로 모든 실외 등 추적기에서 불안정을 일으킬 수 있습니다. 예를 들어 겨울 및 여름에 동일한 공간에 대 한 추적 외부 secondhand light 연도의 서로 다른 시간에 더 높은 될 수 있으므로 현저 하 게 서로 다른 결과 생성할 수 있습니다.

안정적인 500-1000 lux는 luxmeter, 있는 경우 시작 하는 것이 좋습니다. 

#### <a name="types-of-lighting"></a>조명의 유형
다양 한 유형의 공간에서 조명 추적에도 영향을 줄 수 있습니다. 전구는 경우 AC 빈도 50hz, 광원 50hz에 펄스를 보내는 것은 통해 실행 AC 전기를 사용 하 여 펄스입니다. 사람이이 펄스 발견 되지 합니다. 그러나 HoloLens' 30fps 카메라 이러한 변경 내용을 확인-일부 프레임이 밝고, 일부 잘못 켜지, 되며 일부 과도 하 게 제공 될 카메라 밝은 펄스 보완 하려고 합니다.

미국의 전기 빈도 표준 이므로 60hz, HoloLens' 프레임 속도 사용 하 여 전구 펄스는 아니기 때문-60hz 펄스 Hololens' 30FPS 프레임 속도 맞춰집니다. 그러나 여러 국가 AC 빈도 표준 즉, 일부 Hololens 프레임 중 펄스에서 수행 될 하 고 일부는 그렇지 않습니다 50hz의 경우 특히 유럽에서 형광등 문제와 알려져 있습니다. 

몇 가지 방법으로 깜박이 문제를 해결 하는 데 할 수 있습니다. 온도, 전구 수명 및 준비 주기는 형광 깜박임의 일반적인 원인 및 전구 대체 도움이 될 수 있습니다. 전구를 강화 하 고 현재 그립니다 상수 되는지도 도움이 됩니다. 

### <a name="items-in-a-space"></a>공간에 대 한 항목
HoloLens에서는 고유 환경 랜드마크, 라고도 *기능*, 공간에서 자신을 찾으려고 합니다. 

장치에 공간에는 것을 알 수 없습니다 장치 기능 저하 영역에서 거의 발생 하지 않는다는 추적할 수 있습니다. 공간의 벽에 기능 추가 일반적으로 추적을 개선 하는 좋은 방법입니다. 포스터 기호 테이프에 기록 된 벽, 공장, 고유 개체 또는 다른 유사한 항목을 모든 도움말입니다. 복잡 센터는 적절 한 추적을 발생 시키는 environment의 좋은 예-단일 영역에 다른 기능이 많이 있습니다. 

또한 동일한 공간에 고유한 기능을 사용 합니다. 동일한 포스터 벽을 여러 번 반복 예를 들어는 혼동을 일으킬 장치 처럼는 HoloLens 반복적인 포스터에서 확인 하는 것 중 알 수 없습니다. 고유한 기능을 추가 하는 일반적인 방법 중 하나를 만드는 고유 마스킹 테이프 줄 벽 및 공간의 floor 따라 nonrepetitve 패턴을 사용 하는 것입니다. 

좋은 질문을 자신에 게 질문 기능-하면 고유 하 게 찾을 수 직접 공간에 적은 양의 장면 방금 확인 한 경우 그렇지 않은 경우 장치에서 문제가 발생 추적도 가능성이 높습니다.

#### <a name="wormholes"></a>웜 홀
두 개의 영역 또는 동일한 모양을 유지 하는 지역에 있는 동일 하다는 추적기 생각할 수 있습니다. 생각 하는 것을 속여 자체 장치에서이 결과 다른 곳입니다. 이러한 유형의 반복적인 영역 이라고 *웜 홀*합니다. 

웜 홀을 방지 하려면 동일한 공간에 동일한 영역을 방지 하려고 합니다. 동일한 영역 팩터리 스테이션, 건물, 서버 랙 또는 워크스테이션에서 windows를 포함할 경우에 따라 수 있습니다. 영역 레이블 또는 각 유사한 영역에 고유한 기능을 추가 합니다. 웜 홀을 완화 시킬 수 있습니다.
 
### <a name="temporal-stability-of-a-space"></a>공간의 temporal 안정성
환경을 지속적으로 변화 하 고 변경 하는 경우 장치에 대해 찾을 안정적인 기능이 없습니다. 

공간에서는 쉽습니다 추적 손실 사람을 비롯 한 된 개체가 더 많습니다. 이동 컨베이어 벨트, 항목 생성의 여러 상태 및 많은 사람이 공간에 있는 모든 것으로 알려져 추적 문제를 일으킬 합니다.
 
### <a name="proximity-of-the-user-to-items-in-the-space"></a>사용자의 공간에는 항목에 근접
마찬가지로 개체 눈 가까이 제대로 사람은 없습니다 집중 하는 방법에 HoloLens 좌충우돌 개체의 카메라에 가까워질 때. 개체가 너무 가까이 모두 카메라를 사용 하 여 볼 수, 하나의 카메라를 차단 하는 개체는 장치 개체에 대해 추적을 사용 하 여 훨씬 더 많은 문제를 해야 합니다. 

카메라 개체에서 15 cm 보다 더 가깝게 볼 수 있습니다.
 
### <a name="surfaces-in-a-space"></a>공간에서 화면
강력한 반사 화면 모양이 가능성이 추적에 영향을 주는 각도 따라 달라 집니다. 주변을 이동 하면 빛을 반영 하며 이동 하면 화면에서 다른 개체를 표시 하는 경우 새로운 자동차-생각할 수 있습니다. 추적기를 다른 개체는 변화 하는 환경 화면 나타냅니다 반영 논리곱 장치 추적 합니다.

Shiny 작은 개체는 추적에 대해 하기 쉽습니다.

### <a name="wifi-fingerprint-considerations"></a>WiFi 지문 고려 사항
WiFi를 사용으로 지도 데이터 상호 연관 됩니다 WiFi 지문 실제 WiFi 네트워크/라우터에 연결 되지 않았어도 합니다. WiFi 정보 없이 공간과 홀로그램 인식 하도록 약간 느려질 수 있습니다. WiFi 신호 크게 변경 하는 경우 장치는 완전히 다른 공간에는 생각할 수 있습니다.

식별 네트워크 (즉, SSID, MAC 주소) Microsoft 및 참조에는 HoloLens 로컬 보관 될 모든 WiFi로 전송 되지 않습니다.

홀로그램을 배치 하는 데 문제가 있는 경우 HoloLens 자세히 매핑할 수 있도록 공간을 따라 이동 하십시오. 

에 HoloLens 공간에 매핑할 수 없습니다 또는 보정을 벗어났습니다 Limited 모드를 입력할 수 있습니다. 제한 된 모드에서 홀로그램 환경에 적용할 수 없습니다.

## <a name="environment-management"></a>환경 관리
"정리" 할 수 있도록 하는 두 설정이 홀로그램 및 원인 HoloLens 공백을 "잊도록" 합니다.  "홀로그램 및 환경"에 있는 설정 앱도 설정 앱에서 "개인 정보" 아래에 나타나는 두 번째 설정을 사용 하 여 합니다.

1.  홀로그램 주변 삭제 –이 설정은 선택 하 여 모든 고정된 홀로그램 및 장치를 찾으면 "현재 공간"에 대 한 모든 저장 된 지도 데이터 HoloLens 지우기 됩니다.  새 지도 섹션을 만들고 홀로그램을 같은 영역에 다시 배치 되 면 해당 위치에 대 한 데이터베이스에 저장 합니다.

2.  모든 홀로그램 삭제 –이 설정은 선택 하 여 HoloLens 지도 데이터가 모두 지워집니다 및 고정 된 공간의 전체 데이터베이스에 제공 합니다.  홀로그램 없습니다를 다시 검색할 하 고 모든 홀로그램 다시 지도 섹션에서는 데이터베이스에 저장 하려면 새로 배치 해야 합니다.

### <a name="managing-your-spaces"></a>사용자 공간 관리

HoloLens 장치에 로컬로 저장 된 단일 데이터베이스로 지도 섹션 및 서로 다른 공간을 축소 되어 있습니다. 맵 데이터베이스 에서만 사용할 수 있습니다는 내부 시스템을 사용자 장치에 적용 되지 않고 PC에 연결 및/또는 파일 탐색기 앱을 사용 하는 경우에 액세스할 수 있는 안전 하 게 저장 됩니다. Bitlocker가 활성화 저장된 지도 데이터가 암호화 됩니다.

맵 구성 요소를 여러 홀로그램 위치/홀로그램 간의 연결 경로가 없는 다른 위치에 저장 하는 경우 존재 합니다.  동일한 지도 섹션 내에서 고정 홀로그램 "주변"는 현재 공간으로 간주 됩니다.

"현재 공간"의 작은 하위 집합을 내보낼 API 개발자는 (현재 인식할 수 있는 맵 구성 요소 부분) 공유 홀로그램 시나리오를 사용 하도록 설정 합니다.  현재 있는지 매핑된 모든 공간의 전체 데이터베이스를 다운로드 하는 메커니즘이 없습니다.

### <a name="calibration"></a>보정

프로그램 홀로그램 흔들리거나 흔들리는 찾거나 홀로그램을 배치 하는 데 문제가 있는 경우 가장 먼저 시도 하는 경우는 [보정 앱](calibration.md)합니다. 이 앱에 HoloLens를 사용 하는 동안 모든 뜨거움을 발생 하는 경우에 도움이 됩니다.

보정 앱에 연결할 설정으로 이동 > 시스템 > 유틸리티입니다. 열기 보정을 선택 하 고 지침을 따릅니다.

보정 앱을 실행 하 고 여전히 문제가 있는 홀로그램 품질, 또는 잦은 "손실 추적" 메시지가 표시 되는 경우 센서 튜닝 앱을 사용해 보세요. 설정으로 이동 > 시스템 > 유틸리티를 열고 센서 튜닝을 선택 하 고 지침을 따릅니다.

누군가가 사용 하 여 HoloLens를 하려는 경우 해야 앱을 실행 하기 보정 먼저 장치가 제대로 설정 되어에 있도록 합니다.
