# <a name="testing-your-app-on-hololens"></a>HoloLens에 앱 테스트

테스트 HoloLens 응용 프로그램은 Windows 응용 프로그램 테스트와 매우 비슷합니다. 모든 일반적인 영역 (기능, 상호 운용성, 성능, 보안, 안정성, 등) 고려해 야 합니다. 그러나가 특수 처리 또는 PC 또는 phone 앱에서 일반적으로 준수 하지 않을 수 있는 세부 정보에 주의 해야 하는 몇 가지 영역입니다. Holographic 앱 다양 한 환경에서에서 원활 하 게 실행 해야 합니다. 또한 항상 성능 및 사용자 편안 하 게 유지 해야 합니다. 이러한 영역을 테스트 하는 것에 대 한 지침은이 항목에서 자세히 나와 있습니다.

## <a name="performance"></a>성능

Holographic 앱 다양 한 환경에서에서 원활 하 게 실행 해야 합니다. 또한 항상 성능 및 사용자 편안 하 게 유지 해야 합니다. 성능은 전체 항목을 사용된 했으므로 Holographic 앱으로 사용자의 환경에 매우 중요 합니다. 읽기 및 수행 했는지 확인 하십시오는 [혼합 현실에 대 한 성능 이해](understanding-performance-for-mixed-reality.md)

## <a name="testing-3d-in-3d"></a>3d에서 3D 테스트
1. **가능한 만큼 서로 다른 공간에 앱을 테스트 합니다.** 큰 방, 작은 방, 욕실, 사용, 방, 사무실, 등을 시도 합니다. 또한 비 세로 벽, 곡선된 벽 아닌 가로 최대값이 같은 비표준 기능을 사용 하 여 고려 방 야 합니다. 작동 방 간에 전환 되 고, 경우에 바닥, 복도 또는 계단 거치지?
2. **다른 조명 조건에서 앱을 테스트 합니다.** 응답 하나요 제대로 조명, 검은 화면, 투명 한/반사 화면 미러, 투명 벽 등과 같은 여러 환경 조건입니다.
3. **다른 동작 조건에서 앱을 테스트 합니다.** 장치에 놓고 동작의 다양 한 상태에서 시나리오를 시도 합니다. 응답 하나요 제대로 다른 이동 또는 안정적인 상태로?
4. **다른 관점에서에서 앱을 작동 하는 방법을 테스트 합니다.** 잠긴 world 홀로그램 경우 그 뒤에 사용자를 안내 하는 경우는 것이 되나요? 항목에서 사용자를 홀로그램와 제공 되는 경우 어떻게 되나요? 그렇다면 사용자에서 위 또는 아래 홀로그램 살펴봅니다?
5. **공간 및 오디오 큐를 사용 합니다.** 앱이를 사용 하 여 사용자를 손실 하지 못하도록 해야 합니다.
6. **서로 다른 수준의 주변 소음에 앱을 테스트 합니다.** 음성 명령에 구현한 경우 다양 한 수준의 주변 소음을 사용 하 여 호출 시도 합니다.
7. **장착 하는 앱과 고정 테스트**합니다. 좌석 배정 하 고 고정 위치 모두에서 테스트 해야 합니다.
8. **다른 거리에서 앱 테스트**합니다. UI 요소 읽고 수 멀리 떨어진 곳에서 상호 작용할? 사용자 앱 react에 너무 가까이 시작 하는 사용자에 게는 사용자 제공?
9. **일반적인 앱 표시줄 상호 작용에 대해 앱 테스트**합니다. 모든 앱 타일 및 2D 유니버설 앱에는 [앱 바](navigating-the-windows-mixed-reality-home.md#moving-and-adjusting-apps) 혼합 환경에서 앱을 배치 하는 방식을 제어할 수 있습니다. 앱 프로세스를 정상적으로 종료 제거를 클릭 하 고 2D 유니버설 앱의 컨텍스트 내에서 뒤로 단추를 지원 하는지 확인 합니다. Try 크기 조정 및 앱에서 이동 [조정 모드](navigating-the-windows-mixed-reality-home.md#moving-and-adjusting-apps) 활성화 된 동안 및 일시 중단 된 앱 타일을 것입니다.

## <a name="comfort"></a>편안 함
1. **평면을 자릅니다.** 위치에 주의할 수 있도록 수 [홀로그램 렌더링](hologram-stability.md#hologram-render-distances)합니다.
2. **실제 헤드 이동 일치 하지 않습니다. 가상 이동을 하지 마십시오.** 사용자의 실제 동작의 대표 하지 않은 방식으로 카메라를 이동 하지 않습니다. 장면을 통해 사용자를 이동 해야 하는 앱, 예측 가능한 동작을 확인, 가속, 최소화 및 움직임을 제어 하도록 허용할 합니다.
3. **홀로그램 품질 지침을 따릅니다.** 구현 하는 고성능 앱을 [홀로그램 품질 지침](hologram-stability.md) 사용자 뜨거움 발생할 확률이 낮아집니다.
4. **세로가 아닌 가로로 홀로그램을 배포 합니다.** 사용자가 지출 조회 하는 기간 동안 확장 또는 축소는 목에 피로를 발생할 수 있습니다.

## <a name="input"></a>입력

### <a name="gaze-and-gestures"></a>응시 및 제스처

[Gaze](gaze.md) 환경과 홀로그램 것을 목표로 하는 데 사용할 수 있는 HoloLens에 입력 하는 기본 양식입니다. 볼 수 있습니다 시각적으로 응시 프로그램을 대상으로 하는 커서 위치에 따라. 것 게이즈 커서 마우스 커서를 사용 하 여 연결에 공통적으로 적용 합니다.

[제스처](gestures.md) 는 마우스 클릭 같은 홀로그램의 상호 작용 합니다. 대부분의 마우스 및 터치 동작, 같지만 이해 하 고 서로 때 유효성을 검사 해야 합니다.

**앱에 마우스 및 터치를 사용 하 여 다른 동작을 확인 합니다.** 불일치 식별 되 고 사용자가 더 자연 스러운 환경이 있게 한 설계 결정에 도움이 됩니다. 예를 들어 트리거 hover에 따라 작업 합니다.

> [!NOTE]
> HoloLens 2 관련 된 자세한 지침 [예정](index.md#news-and-notes)합니다.

### <a name="custom-voice-commands"></a>사용자 지정 음성 명령

[입력 음성](voice-input.md) 는 자연 스러운 형태의 상호 작용 합니다. 사용자 환경 마법 또는 원하는 명령 및 노출 하는 방법에 따라 혼동을 줄 수 있습니다. 일반적으로 사용자 지정 명령으로 "Select" 또는 "Hey Cortana"와 같은 음성 명령 시스템을 하지 사용 해야 합니다. 고려할 몇 가지 사항은 다음과 같습니다.
1. **비슷하게 들리는 명령을 사용 하지 마십시오.** 잠재적으로 잘못 된 명령을 트리거할 수 있습니다이 있습니다.
2. **가능한 경우 획 다양 한 단어를 선택 합니다.** 이 최소화 및/또는 false 활성화 방지 합니다.

### <a name="peripherals"></a>주변 장치

통해 앱에 대 한 대화형 [주변 장치](hardware-accessories.md)합니다. 하지만 앱을 확인해 두는 몇 가지 방법이 있습니다 해당 기능을 활용 하기 위해 특별 한 모든 작업을 수행할 필요가 없습니다.
1. **사용자 지정 상호 작용의 유효성을 검사 합니다.** 같은 앱에 대 한 사용자 지정 바로 가기 키입니다.
2. **입력된 형식 전환의 유효성을 검사 합니다.** 여러 입력된 방법을 사용 하 여 음성, 제스처, 마우스 및 키보드 동일한 시나리오를 모두 같은 작업을 완료 하려고 합니다.

## <a name="system-integration"></a>시스템 통합

### <a name="battery"></a>배터리

전원 배터리를 드레이닝하 얼마나 신속 하 게 이해에 연결 하지 않고 응용 프로그램을 테스트 합니다. 전원 LED 판독값을 보면 배터리 상태를 쉽게 파악할 수 있습니다 하나. ![배터리를 나타내는 LED 상태](images/batterypowerledindication-500px.png)

### <a name="power-state-transitions"></a>전원 상태 전환

전원 상태 간에 전환 될 때 예상 대로 주요 시나리오 작업의 유효성을 검사 합니다. 원래 위치에서 응용 프로그램 유지 하는 예를 들어, 상태를 올바르게 유지 않습니다 것? 계속 예상 대로 작동 하는?
1. **대기/다시 시작 합니다.** 대기 모드를 입력 하려면 키를 누릅니다 하 고 전원 단추를 즉시 해제 수 있습니다 하나. 또한 장치 3 분 후 자동으로 대기 모드를 입력 합니다. 대기 모드에서 다시 시작 하려면 키를 누릅니다 하 고 전원 단추를 즉시 해제 하나입니다. 장치 연결 하거나 전원을에서 분리 하는 경우에 계속 됩니다.
2. **종료 / 다시 시작 합니다.** 종료, 키를 누른 전원 단추를 지속적으로 6 초입니다. 를 다시 시작 하려면 전원 단추를 누릅니다.

## <a name="emulator-and-simulated-input"></a>에뮬레이터 및 시뮬레이션 된 입력

합니다 [HoloLens 에뮬레이터](using-the-hololens-emulator.md) 를 매우 효율적으로 다양 한 시뮬레이션 된 사용자의 특성 및 공백을 사용 하 여 Holographic 앱을 테스트 하기 편리 합니다. 몇 가지 제안을 효과적으로 앱을 테스트 하려면 에뮬레이터를 사용 하는 다음과 같습니다.
1. **에뮬레이터의 가상 방을 사용 하 여 테스트를 확장 합니다.** 에뮬레이터는 더 많은 환경에서 앱을 테스트 하는 데 사용할 수 있는 가상 방 집합이 함께 제공 됩니다.
2. **에뮬레이터를 사용 하 여 모든 각도에서 앱을 살펴봅니다.** PageUp/PageDn 키를 늘리거나 줄여서 시뮬레이션 된 사용자를 확인 합니다.
3. **실제 HoloLens 사용 하 여 앱을 테스트 합니다.** HoloLens 에뮬레이터에는 앱에 대해 반복 하 고 새 버그를 catch 하지만 테스트 해야 실제 HoloLens에 Windows 스토어에 제출 하기 전에 신속 하 게 돕는 뛰어난 도구입니다. 이 성능 및 환경 실제 하드웨어에서 훌륭한 되는지 확인 해야 합니다.

## <a name="automated-testing-with-perception-simulation"></a>Perception 시뮬레이션을 사용 하 여 테스트 자동화

일부 앱 개발자가 앱의 테스트를 자동화 하려고 할 수 있습니다. 간단한 단위 테스트를 넘어 사용할 수 있습니다 합니다 [perception 시뮬레이션](perception-simulation.md) 앱에 대 한 사람과 world 입력을 자동화 하는 HoloLens의 스택. Perception 시뮬레이션 API HoloLens 에뮬레이터 또는 실제 HoloLens로 시뮬레이션 된 입력을 보낼 수 있습니다.

## <a name="windows-app-certification-kit"></a>Windows 앱 인증 키트

앱의 최상의 기회 수 있도록 [Windows 스토어에 게시 된](submitting-an-app-to-the-microsoft-store.md), 유효성 검사 및 인증을 위해 제출 하기 전에 로컬로 테스트 합니다. 앱이 Windows.Holographic 장치 제품군을 대상으로 하는 경우는 [Windows 앱 인증 키트](https://msdn.microsoft.com/library/windows/apps/xaml/mt186449.aspx) PC에서 로컬 정적 분석 테스트만 실행 됩니다. 에 HoloLens에서 테스트가 실행 됩니다.
