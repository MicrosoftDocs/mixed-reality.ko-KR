# <a name="shared-experiences-in-mixed-reality"></a>혼합된 현실 환경 공유했습니다

홀로그램 하나만 사용자에 게 개인 유지할 필요가 없습니다. Holographic 앱에서 공유할 수 있습니다 [공간 앵커](coordinate-systems.md) 하나 HoloLens, iOS 또는 Android 장치를 다른에서 여러 장치에서 실제로 같은 위치에서 홀로그램을 렌더링 하는 사용자를 사용 하도록 설정 합니다.

## <a name="six-questions-to-define-shared-scenarios"></a>공유 시나리오를 정의 하는 6 개 질문

공유 경험에 대 한 디자인을 시작 하기 전에 대상 시나리오를 정의 하는 것이 반드시 합니다. 이러한 시나리오 명확히 새로운 디자인 하 고 설정 하는 데 일반적인 어휘를 비교 하 고 환경에서 필요한 기능을 대조 합니다. 이 새 미디어에 내재 된 민감한 기회 키인 core 문제 및 솔루션에 대 한 다른 경로 이해 합니다.

내부 프로토타입 및 우리의 HoloLens 파트너 기관의 탐색을 통해 공유 시나리오를 정의 하는 데 도움이 되는 6 개의 질문이 만들었습니다. 이러한 질문에는 시나리오의 중요 한 특성을 추출 하는 데 완벽 한 아니며 프레임 워크를 형성 합니다.

* **프레젠테이션**: 경우 동일한 콘텐츠는 여러 사용자에 게 표시 됩니다. 예를 들어 다음과 같은 가치를 제공해야 합니다. 교수 강의 모든 사용자에 게 제공 되는 동일한 holographic 자료를 사용 하 여 여러 학생 들에 게 제공 됩니다. 그러나 조언을 자신의 고유한 힌트 및 다른 사용자가 볼 수 있는 정보에 있을 수 있습니다.

## <a name="get-started-sharing-experiences"></a>경험을 공유 시작

응용 프로그램 및 시나리오에 따라 원하는 환경을 달성 하기 위해 다양 한 요구 사항의 없을 것입니다. 이러한 일부 포함
* 일치 항목 만들기: 세션을 만들고, 세션을 보급 하 고 검색 모두 로컬로 특정 피플 초대 및 원격 세션에 연결할 수 있습니다.
* 공유를 고정 합니다. 홀로그램 모든 사람에 대 한 동일한 위치에 표시 되도록 공통 로컬 공간에서 여러 장치에서 좌표를 맞출 수 있습니다.
* 네트워킹: 상호 작용 하는 기능을 배치 하 고 동기화 된 사용자 및 홀로그램 이동 모든 참석자에 대해 실시간입니다.
* 상태 저장: 세션 중 조인에 대 한 공간에 홀로그램 특성 및 위치를 저장, 이후 시간 및 네트워크 문제에 대 한 견고성 회수할 수 있습니다.


공유 경험을 키는 여러 사용자가 자신의 장치에서 장치의 좌표에 맞게 앵커를 공유 하 여 자주 수행 전 세계에서 동일한 홀로그램을 표시 합니다.
앵커를 공유 하려면 사용 합니다 <a href="https://docs.microsoft.com/azure/spatial-anchors" target="_blank">Azure 공간 앵커</a>:
* 먼저 사용자를 홀로그램을 배치합니다.
* 앱을 만듭니다는 [공간 앵커](coordinate-systems.md) 전 세계에서 정확 하 게 해당 홀로그램을 고정 합니다.
* 앵커를 HoloLens, iOS 및 Android 장치를 통해 공유할 수 있습니다 합니다 <a href="https://docs.microsoft.com/azure/spatial-anchors" target="_blank">Azure 공간 앵커</a>합니다. 

공유 공간 앵커를 사용 하 여 각 장치에서 앱 설정 되었습니다 일반적인 좌표계는 콘텐츠를 배치할 수 있습니다. 이제 앱은 위치와 동일한 위치에 홀로그램 방향을 지정을 확인할 수 있습니다.
HoloLens 장치에 대 한 앵커 오프 라인으로 하나의 장치에서 다른도 공유할 수 있습니다.  아래 링크를 사용 하 여 응용 프로그램에 대 한 최적의 항목이 결정 합니다.
