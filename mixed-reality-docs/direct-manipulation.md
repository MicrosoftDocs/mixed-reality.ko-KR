# <a name="direct-manipulation-with-hands"></a>수동으로 직접 조작
직접 조작은 홀로그램을 손으로 직접 터치하게 되는 입력 모델입니다. 직접 조작을 사용하여 목적은 개체가 실제 환경에서와 마찬가지로 동작하게 하는 것입니다. 단추는 간단히 눌러서 활성화하고, 개체는 잡아서 선택할 수 있으며, 2D 콘텐츠는 가상의 터치 스크린처럼 동작합니다.  이때문에 직접 조작은 사용자가 학습하기 쉽고 재미있습니다.  이 방식은 "가까운" 입력 모델로 간주됩니다. 즉, 팔을 뻗어서 닿을 수 있는 콘텐츠와 상호 작용하는 데 가장 적합합니다.

직접 조작은 어포던스를 기준으로 합니다. 즉, 사용자 친화적입니다. 사용자를 교육하기 위한 기호화된 제스처는 없습니다. 모든 상호 작용은 사용자가 터치하거나 잡을 수 있는 시각적 요소를 중심으로 구축됩니다.

## <a name="device-support"></a>장치 지원


| 입력 모델 | [HoloLens(1세대)](hololens-hardware-details.md) | HoloLens 2 |[몰입형 헤드셋](immersive-headset-hardware-details.md)|
|:-------- | :-------| :--------| :------------|
| 수동으로 직접 조작 | ❌ 지원되지 않음 | ✔️ 권장 | ➕ 대안으로 [손으로 가리키고 커밋](point-and-commit.md)이 권장됩니다.

직접 조작은 HoloLens 2의 기본 입력 모델로, 새롭게 연결된 손 추적 시스템을 활용합니다. 이 입력 모델은 모션 컨트롤러를 사용하여 몰입형 헤드셋에서도 사용할 수 있지만 개체 조작 이외의 상호 작용을 위한 기본 수단으로는 권장되지 않습니다.  HoloLens(1세대)에서는 직접 조작을 사용할 수 없습니다.


## <a name="collidable-fingertip"></a>충돌 가능한 손끝

HoloLens 2에서 사용자의 실제 손은 왼손 및 오른손 골격 모델로 인식 및 해석됩니다. 손으로 홀로그램을 직접 터치한다는 개념을 적절히 구현하기 위해 각 손 골격 모델의 5개 손끝에 5개의 충돌체(collider)를 연결할 수 있습니다. 그러나 실질적으로 촉각 피드백이 부족하기 때문에 10개의 충돌 가능한 손끝이 홀로그램과 예기치 않은 충돌을 유발했습니다. 

따라서 각 집게 손가락에만 충돌체(collider)를 추가하는 것이 좋습니다. 아래 이미지와 같이, 충돌 가능한 집게 손끝은 여전히 다른 손가락과 관련된 다른 터치 제스처(예: 한 손가락으로 누르기, 한 손가락으로 탭하기, 두 손가락으로 누르기, 다섯 손가락으로 누르기)의 활성 터치 포인트 역할을 할 수 있습니다.

![충돌 가능한 손끝 이미지](images/Collidable-Fingertip-720px.jpg)

### <a name="sphere-collider"></a>구형 충돌체(collider)

근거리 타기팅을 위한 더 나은 신호를 제공하려면 임의의 일반 셰이프를 사용하는 대신, 구형 충돌체(collider)를 사용하고 시각적으로 렌더링하는 것이 좋습니다. 터치 정확도를 높이려면 구의 지름이 집게 손가락의 두께와 일치해야 합니다. 손 API 호출하여 손가락 두께 변수를 쉽게 검색할 수 있습니다.

### <a name="fingertip-cursor"></a>손끝 커서

집게 손끝에 충돌 가능한 구를 렌더링하는 것 외에, 더 나은 대화형 근거리 타기팅 환경을 얻기 위해 고급 솔루션인 손끝 커서를 만들었습니다. 집게 손끝에 연결된 도넛형 커서입니다. 아래에 설명된 것처럼, 근접성에 따라 대상의 방향 및 크기 변화에 동적으로 반응합니다.

* 집게 손가락을 홀로그램 쪽으로 움직이면 커서는 항상 홀로그램 표면에 평행하게 움직이며 크기가 서서히 축소됩니다.
* 손가락으로 표면을 터치하는 즉시, 커서가 점으로 축소되고 터치 이벤트를 내보냅니다.

대화형 피드백을 사용하면 아래와 같이 웹 콘텐츠의 하이퍼링크 트리거 또는 단추 누르기와 같은 높은 정확도의 근거리 타기팅 작업을 달성할 수 있습니다. 

![손끝 커서 이미지](images/Fingertip-Cursor-720px.jpg)

## <a name="bounding-box-with-proximity-shader"></a>근접 셰이더가 있는 경계 상자

홀로그램 자체에는 촉각 피드백 부족을 보정하기 위해 시각적 및 청각적 피드백을 둘 다 제공하는 기능도 필요합니다. 이를 위해 근접 셰이더를 포함하는 경계 상자 개념을 생성합니다. 경계 상자는 3D 개체를 둘러싸는 최소 체적 영역입니다. 경계 상자에 근접 셰이더라는 대화형 렌더링 메커니즘을 포함합니다. 근접 셰이더는 다음과 같이 동작합니다.

* 집게 손가락이 범위 내에 있는 경우 손끝 스포트라이트가 경계 상자 표면에 캐스팅됩니다.
* 손끝이 화면에 가까워질수록, 스포트라이트도 그에 따라 압축됩니다.
* 손끝이 표면을 터치하는 즉시, 전체 경계 상자의 색상이 바뀌거나 터치 상태를 반영하는 시각적 효과가 생성됩니다.
* 한편 시각적 터치 피드백을 향상시키기 위해 음향 효과를 활성화할 수 있습니다.

![근접 셰이더가 있는 경계 상자 이미지](images/Bounding-Box-With-Proximity-Shader-720px.jpg)

## <a name="pressable-button"></a>누를 수 있는 단추

충돌 가능한 손끝을 사용하여, 매우 기본적인 홀로그래픽 UI 구성 요소인 누를 수 있는 단추와 상호 작용할 준비가 되었습니다. 누룰 수 있는 단추는 직접적인 손가락으로 누르기에 맞게 조정된 홀로그래픽 단추입니다. 다시 말하지만, 촉각 피드백이 부족하기 때문에 누를 수 있는 단추에는 촉각 피드백 관련 이슈를 해결하기 위한 몇 가지 메커니즘이 포함되어 있습니다.

* 첫 번째 메커니즘은 이전 섹션에서 자세히 설명하는 근접 셰이더가 있는 경계 상자입니다. 이 메커니즘은 사용자가 단추를 사용해서 접근하고 접촉할 수 있도록 하기 위해 더 나은 근접 감각을 제공합니다.
* 두 번째 메커니즘은 누르기입니다. 손끝이 단추에 닿은 후에는 누르기 감각이 생성됩니다. 이 메커니즘에서는 단추가 깊이 축을 따라 손끝과 함께 움직입니다. 단추는 지정된 깊이(누를 때)에 도달하거나 해당 깊이를 통과한 후 떠나면(뗄 때) 트리거될 수 있습니다.
* 단추가 트리거될 때 피드백을 향상시키기 위해 음향 효과를 추가해야 합니다.

![누를 수 있는 단추 이미지](images/Pressable-Button-720px.jpg)

## <a name="2d-slate-interaction"></a>2D 슬레이트 상호 작용

2D 슬레이트는 웹 브라우저와 같은 2D 앱 콘텐츠를 호스트하는 홀로그램 컨테이너입니다. 직접 조작을 통해 2D 슬레이트와 상호 작용하기 위한 디자인 개념은 실제 터치 스크린과 상호 작용하는 상상 모델을 활용하는 것입니다.

슬레이트 접촉 부분과 상호 작용하려면

* 집게 손가락으로 하이퍼링크 또는 단추를 누릅니다.
* 집게 손가락으로 슬레이트 콘텐츠 위/아래로 스크롤합니다.
* 사용자는 2개의 집게 손가락을 사용하여 손가락의 상대적 모션에 따라 슬레이트 콘텐츠를 확대 및 축소합니다.

![2D 슬레이트 이미지](images/2D-Slate-Interaction-720px.jpg)

2D 슬레이트 자체 조작:

* 손을 모서리 및 가장자리 쪽으로 가져가 가장 가까운 조작 어포던스를 표시합니다.
* 조작 어포던스를 잡고, 모서리 어포던스를 통해 균일하게 크기를 조정하고, 가장자리 어포던스를 통해 재배치합니다.
* 2D 슬레이트의 맨 위에 있는 홀로바를 잡아 전체 슬레이트를 이동할 수 있습니다.

![슬레이트 조작 이미지](images/Manipulate-2d-slate-720px.jpg)

## <a name="3d-object-manipulation"></a>3D 개체 조작

HoloLens 2에서는 각 3D 개체에 경계 상자를 적용하여 손으로 3D 홀로그래픽 개체를 직접 조작할 수 잇습니다. 경계 상자는 근접 셰이더를 통해 더 나은 깊이 인식을 제공합니다. 경계 상자를 사용하면 3D 개체 조작을 위해 두 가지 디자인 방식을 사용할 수 있습니다.

### <a name="affordance-based-manipulation"></a>어포던스 기반 조작

이 방식에서는 경계 상자와 주변의 조작 어포던스를 통해 3D 개체를 조작할 수 있습니다. 사용자의 손이 3D 개체에 가까워지는 즉시, 경계 상자와 가장 가까운 어포던스가 표시됩니다. 사용자는 경계 상자를 잡아 전체 개체를 이동하고, 가장자리 어포던스를 적용하여 회전하고, 모서리 어포던스를 적용하여 균일하게 크기를 조정할 수 있습니다.

![3D 개체 조작 이미지](images/3D-Object-Manipulation-720px.jpg)

### <a name="non-affordance-based-manipulation"></a>비어포던스 기반 조작

이 메커니즘에서는 경계 상자에 어포던스가 연결되지 않습니다. 사용자는 경계 상자만 표시한 후 직접 상호 작용할 수 있습니다. 한 손으로 경계 상자를 잡을 경우 개체의 변환 및 회전은 손의 동작 및 방향과 연결됩니다. 두 손으로 경계 상자를 잡을 경우 두 손의 상대적 동작에 따라 변환, 크기 조정 및 회전할 수 있습니다.

특정 조작에는 정밀도가 요구되므로, 더 높은 세분성을 제공하는 **어포던스 기반 조작**을 사용하는 것이 좋습니다. 유연한 조작을 위해서는 즉각적이면서 유용한 환경을 지원하는 **비어포던스 조작**을 사용하는 것이 좋습니다.

## <a name="instinctual-gestures"></a>직관적 제스처

HoloLens(1세대)에서는 블룸 및 에어 탭과 같은 미리 정의된 몇 가지 제스처를 사용자에게 학습했습니다. HoloLens 2에서는 사용자에게 기호화된 제스처를 기억하도록 요구하지 않습니다. 사용자가 홀로그램 및 콘텐츠와 상호 작용하는 데 필요한 모든 사용자 제스처는 직관적입니다. 직관적 제스처를 달성하려면 사용자가 UI 어포던스의 디자인을 통해 제스처를 수행하도록 유도합니다.

예를 들어, 사용자가 두 손가락 모으기로 개체나 제어점을 잡도록 하려면 개체나 제어점이 작아야 합니다. 5개 손가락으로 잡기를 수행하도록 하려면 개체 또는 제어점이 비교적 커야 합니다. 단추의 경우도 마찬가지로, 단추 크기가 작으면 사용자는 한 손가락으로 누르게 되지만, 단추 크기가 크면 손바닥으로 누르게 됩니다.

![](images/Instinctual-Gestures-720px.jpg)

## <a name="symmetric-design-between-hands-and-6-dof-controllers"></a>손과 6 DoF 컨트롤러 간의 대칭 디자인

AR의 손과 VR의 모션 컨트롤러 간에는 상호 작용 평행 이론이 작용한다는 것을 알 수 있습니다. 두 입력은 모두 해당 환경에서 직접 조작을 트리거하는 데 사용할 수 있습니다. HoloLens 2의 근거리에서 손으로 잡아 끄는 동작은 WMR의 모션 컨트롤러에 있는 잡기 단추로 작동하는 것과 같습니다. 따라서 사용자는 두 플랫폼에서 유사하게 조작할 수 있으며, 앱을 한 플랫폼에서 다른 플랫폼으로 이식해도 유용할 수 있습니다.

## <a name="optimize-with-eye-tracking"></a>시선 추적으로 최적화

직접 조작은 의도대로 작동한다면 마법과 같이 느껴질 수 있지만, 손을 움직일 때마다 홀로그램이 의도와 다르게 트리거되면 매우 당황할 수 있습니다.
시선 추적은 사용자의 의도를 보다 잘 식별하는 데 도움이 될 수 있습니다.

* **사용할 때**: 조작 대응을 잘못 트리거하는 경우를 줄입니다. 시선 추적은 사용자가 현재 진행 중인 작업을 더 잘 이해할 수 있도록 합니다.
예를 들어, 실제 작업 공구를 잡기 위해 팔을 뻗으면서 홀로그래픽(지침) 텍스트를 읽는 경우를 가정해 봅니다.

이렇게 하여 이전에 몰랐던 일부 대화형 홀로그래픽 단추 사이에서 실수로 손을 움직이게 됩니다사용자의 FOV(시야갹) 밖에 있을 수도 있음).

  요약: 사용자가 한참 동안 홀로그램을 보지 않았는데도 터치 또는 잡기 이벤트가 감지된 경우, 사용자가 해당 홀로그램과 상호 작용하려는 의도가 실제로 없었을 가능성이 높습니다.

* **선택한 대상**:  또 다른 예로 가양성 활성화를 처리하는 것 외에, 여러 개의 홀로그램이 서로 가깝게 배치된 경우 특히, 사용자 관점에서 정확한 교차점을 알 수 없으므로 잡거나 찌를 홀로그램을 더 잘 식별하는 경우를 들 수 있습니다.

  HoloLens 2의 시선 추적은 시선 응시를 확인할 수 있는 시선 응시의 정확도는 제한되지만, 손 입력으로 조작할 때는 깊이 차이가 있으므로 근거리 조작에 매우 유용할 수 있습니다.  즉, 경우에 따라 조작 위젯을 정확히 잡기 위해 손을 홀로그램 뒤에 둘지 또는 앞에 둘지를 결정하는 것이 어려울 수 있습니다.

* **대상 위치**: 빨리 던지기 제스처에서 사용자가 바라보는 대상에 대한 정보를 사용합니다. 홀로그램을 잡은 후 의도한 대상 쪽으로 대충 던집니다.  

    이러한 조작은 일반적으로 잘 작동하지만, 손 제스처를 빠르게 수행할 경우 대상이 부정확할 수 있습니다. 이러한 경우 시선 추적을 사용하면 손으로 던지기 벡터를 의도한 위치에 다시 연결하는 데 도움이 될 수 있습니다.
