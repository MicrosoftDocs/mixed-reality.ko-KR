# <a name="gestures"></a>제스처

손 제스처 혼합된 현실에서 사용자가 조치를 허용합니다. 상호 작용에 빌드되어 **gaze** 대상으로 하 고 **제스처** 또는 **음성** 모든 요소를 대상으로 지정 된 동작을 수행할 합니다. 손 제스처 공간에서 정확한 위치를 제공 하지 않지만 HoloLens에 콘텐츠를 사용 하 여 즉시 상호 작용을 간단 하 게 다른 보조 프로그램 없이 작업을 시작할 수 있습니다.

## <a name="gaze-and-commit"></a>응시 커밋

작업을 수행 하려면 제스처 사용 하 여 전달 [헤드 게이즈](gaze.md) 메커니즘을 대상으로 합니다. 조합 **Gaze** 하며 **어 탭** 제스처에서 결과 **게이즈 커밋** 상호 작용. 게이즈 커밋 하는 대신 **지점 커밋**에서 사용 가능한 [컨트롤러 동작](motion-controllers.md)합니다. HoloLens에서 실행 되는 앱은만 HoloLens 동작 컨트롤러를 지원 하지 않으므로 게이즈 커밋을 지원 해야 합니다. HoloLens 및 몰입 형 헤드셋 모두에서 실행 되는 앱에 사용자 선택에 게 사용 하 여 입력된 장치에서 제공 게이즈 기반 및 가리키는 기반 상호 작용을 지원 해야 합니다.

## <a name="the-two-core-gestures-of-hololens"></a>HoloLens의 제스처는 두 가지 핵심

HoloLens 현재 두 가지 핵심 구성 요소 제스처-인식 **어 탭** 하 고 **블 룸**합니다. 이러한 두 가지 핵심 상호 작용은 개발자가 액세스할 수 있는 공간 입력된 데이터의 가장 낮은 수준입니다. 다양 한 사용자 작업에 대 한 기초를 형성 합니다.

### <a name="air-tap"></a>어 탭

어 탭 핸드헬드 수직 마찬가지로 마우스 클릭을 사용 하 여 탭 제스처 하거나 선택 합니다. 이 대부분 HoloLens 환경에서 "클릭"에 해당 하는 UI 요소에 대상으로 사용 하 여 후 [Gaze](gaze.md)합니다. 한 번에 대해 알아봅니다 및 모든 앱에서 적용 하는 범용 작업을 것입니다. 단일 단추를 눌러는 다른 방법으로 선택 하는 데는 [HoloLens Clicker](hardware-accessories.md#hololens-clicker) 또는 음성 통화 하 여 "Select" 명령입니다.

![HoloLens에 손 제스처에 대 한 준비 상태](images/image9.png)<br>
*HoloLens에 어 탭에 대 한 준비 상태입니다.*

어 탭에 개별 제스처입니다. 선택 하거나 완료 또는 아니기를 액션 되었거나이 환경 내에서 작성 되지 않습니다. 주요 구성 요소 (예: 두 번 탭 제스처를 단일 탭 보다 다른 항목을 의미)의 조합에서 다른 개별 제스처를 만들려면 몇 가지 특정 목적을 하지 않고 권장 되지는 않지만 수는 있습니다.

![준비 위치에 다음을 탭 하거나 클릭 동작 손가락](images/readyandpress.jpg)<br>
*어 탭을 수행 하는 방법: 준비 위치로 집게 손가락 시키고 손가락 탭 또는 선택 키를 눌러 다음 릴리스를 백업 합니다.*

### <a name="bloom"></a>블 룸

Bloom "home" 제스처는 이며에 대 한 예약 된만 해당 합니다. 다시 시작 메뉴로 이동 하는 데 사용 되는 특수 시스템 동작을 것입니다. 키를 눌러 Windows 키보드 또는 Xbox 컨트롤러에 Xbox 단추는 같습니다. 사용자는 직접을 사용할 수 있습니다.

![HoloLens 블 룸 제스처](images/image10-640px.png)

HoloLens에 블 룸 제스처를 수행 하려면 직접 따로, 함께 손쉽게를 사용 하 여, 야자수 합니다. 그런 다음 직접을 엽니다. 참고로 돌아갈 수 있습니다 항상 시작 "Hey Cortana, Go Home" 라는 가정에서. 앱으로 시스템에 의해 처리 되므로 이러한 홈 작업에 특히 대응할 수 없습니다.

### <a name="gesture-recognizers"></a>제스처 인식기

제스처 인식 사용의 이점 중 하나는 현재 대상된 홀로그램 받아들일 수 제스처에 대 한 제스처 인식기를 구성할 수 있습니다. 플랫폼에는 이러한 특정 지원 되는 제스처를 구분 하는 데 필요한 명확성만 수행 됩니다. 이런 방식으로 방금 어 탭을 지 원하는 홀로그램 임의 길이의 키를 눌러 릴리스 사이의 시간을 허용할 수를 홀로그램 하는 동안는 지원 모두 페이지를 누르고 승격할 수 보류를 눌러서 대기 시간 임계값 후 합니다.

## <a name="hand-recognition"></a>직접 인식

HoloLens 장치에 표시 되는 실습 중 하나 또는 둘 다의 위치를 추적 하 여 손 제스처를 인식 합니다. HoloLens 하나 있는 경우 자동 표시를 **상태를 준비** (뒷면 집게 손가락을 사용 하 여 연결 하면 손 모양 아이콘이) 또는 **누름 상태** (집게 손가락을 사용 하 여 연결 하면 손 모양 아이콘이)의. 다른 동작에는, 하는 HoloLens 무시 됩니다.

각 포인터에 대 한 해당 HoloLens을 감지 (방향) 없이 해당 위치 및 누름된 상태로 액세스할 수 있습니다. 손 모양 아이콘이 제스처 프레임의 가장자리를 따라 가져오려고 다시 HoloLens 볼 수 있는 해당 직접 이동 하는 방법을 알아야 하므로 사용자에 게 표시할 수 있는 방향 벡터를 사용 하 여 제공 됩니다.

## <a name="gesture-frame"></a>제스처 프레임

HoloLens에 제스처를 손 모양 아이콘이 프레임 내 에서"제스처"를 제스처가 감지 카메라 (약 매우 코 접근을 막기 간의 현재, 하)에서 적절 하 게 볼 수 있는 범위에 있어야 합니다. 사용자는 인식 작업의 성공 여부와 자신의 편안 하 게 위해 (여러 사용자가 처음 가정할 제스처 프레임 HoloLens 통해 해당 뷰에서 여야 하며 상호 작용 하기 위해 uncomfortably 해당 무기 거래 보유 하)의이 영역에 학습 해야 합니다. HoloLens Clicker 사용 손 제스처 프레임 내에 있이 필요가 없습니다.

연속 제스처의 경우 특히 위험은 일부 사용자가 원하는 결과 모두 잃으면 서가 게 (예를 들어 일부 holographic 개체를 이동) 하는 동안 중간 제스처에서 제스처 프레임 외부에서 이동 합니다.

고려해 야 하는 방법은 다음 세 가지 있습니다.
* 사용자 교육 제스처 프레임의 존재 여부 및 대략적인 경계 (HoloLens 설치 하는 동안 제공 됩니다).
* 사용자에 게 알리는 경우 해당 제스처는 곧 및 손실된 제스처를 원하지 않는 결과 이어질 수 있는 정도를 응용 프로그램에서 제스처 프레임 경계입니다. 연구에 따르면 이러한 알림 시스템의 핵심 품질을 하 고 HoloLens 셸에서이 유형의 알림 (시각적 개체는 경계에 교차 이루어지는 방향을 나타내는 커서를 중앙에서)의 좋은 예를 제공 합니다.
* 제스처 프레임 경계를 중단에 따른 최소화 해야 합니다. 일반적으로 제스처의 결과 경계에서 중지 되었지만 반대로 수행 되지 않았다는 의미입니다. 예를 들어, 사용자는 대화방에서 일부 holographic 개체 이동 하는 경우 이동 중지할지 제스처 프레임을 초과할 때 하지만 **되지** 시작점을 반환 합니다. 사용자 그런 다음 일부 불만이 발생할 수 있지만 경계를 보다 신속 하 게 이해 및 때마다 해당 전체 의도 된 작업을 다시 시작 하지 않아도 수 있습니다.
